# 调试逻辑规范

## 标准格式

**模式1：最小化设置（推荐）**

```python
# *** 调试逻辑 ***
if __name__ == "__main__":
    配置 = 界面配置()
    def main(page: ft.Page):
        # 不设置任何页面属性，让被测模块自己决定
        page.add(被测模块(配置).render())
    ft.run(main)
```

**模式2：标准设置**

```python
# *** 调试逻辑 ***
if __name__ == "__main__":
    配置 = 界面配置()
    def main(page: ft.Page):
        # 仅设置必要的测试环境属性
        page.padding = 0
        page.bgcolor = 配置.当前主题颜色["bg_primary"]
        page.add(被测模块(配置).render())
    ft.run(main)
```

## 核心原则

- **如实测试**：测试结果必须如实反映被测的类/函数的实际运行情况
- **禁止重置**：不得使用任何参数的重置
- **测试环境设置**：可以设置必要的测试环境属性，但不得影响被测模块内部逻辑
- **只能更改被测模块名称**：其他代码结构保持不变
- **禁止硬编码和额外包装**：不得添加额外的逻辑或包装
- **统一结构**：使用统一的调试逻辑结构

## 测试要求

1. **真实环境**：使用真实的配置和参数，调试逻辑中的任何参数不得第二次覆盖
2. **完整测试**：测试被测模块的所有公开接口
3. **无干扰**：不添加额外的UI重置、布局调整等干扰代码
4. **可复现**：测试结果可以稳定复现
5. **完整参数**：测试的类/模块中必须的默认参数必须完整
   - UI组件必须有默认值（标签、选项等不得为空）
   - 下拉框选项必须包含完整的测试数据
   - 输入框必须有默认值
   - 确保用户可以观察到所有UI元素

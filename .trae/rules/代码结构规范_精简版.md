# 代码结构规范（精简版）

## 核心原则
- **模块完整性**：包含文件头、导入、用户指定变量、类/函数定义、调试逻辑
- **用户变量保护**：AI不得修改用户指定变量区域
- **如实测试**：调试逻辑必须如实反映被测模块运行情况
- **模块隔离**：调用者不得修改被调模块内部实现
- **无冗余**：移除未使用的变量、导入和重复代码

## 模块结构
```python
# -*- coding: utf-8 -*-
"""模块名称：XXX | 设计思路 | 模块隔离原则"""
import flet as ft

# *** 用户指定变量 - AI不得修改 ***
# （用户未指定变量）
# *********************************

class XXX:
    @staticmethod
    def create(param1: str="默认值", param2: int=16) -> ft.Container:
        return ft.Container()

if __name__ == "__main__":
    ft.run(lambda page: page.add(XXX.create()))
```

## 关键规范
1. **文档字符串**：模块级必须包含（模块名称、设计思路、模块隔离原则），禁止包含层级说明
2. **用户指定变量区域**：区域必须存在，用户未指定时标注 `# （用户未指定变量）`；有变量时必须添加中文注释
3. **导入顺序**：标准库 → 第三方库 → 项目内部模块
4. **函数参数**：所有参数应有默认值，`=`两边不加空格
5. **模块隔离**：调用者只通过公开接口访问模块
6. **调试逻辑**：使用统一结构，禁止在调试逻辑中添加初始化代码
7. **代码质量**：函数长度≤20行，类超过300行考虑拆分
8. **禁止冗余别名**：禁止在模块中创建或导出中文别名

# 后端核心开发规范

## 模块划分
```
core/
├── controller/             # 游戏控制
│   ├── game_client.py     # 游戏客户端操作
│   └── actions.py         # 动作组合
├── strategy/               # 决策逻辑
│   ├── base.py            # 策略基类
│   └── scripts/           # 具体脚本
└── utils/                  # 通用工具
    ├── logger.py          # 日志工具
    └── config.py          # 配置管理
```

## 代码质量
- 函数只做一件事，长度≤30行
- 类名避免"-er"/"-or"后缀
- 异常捕获需记录日志，不得静默失败

## 线程安全
- 游戏操作与界面分离，使用队列通信
- 使用`threading.Event`控制脚本启停
- 共享数据需加锁保护

## 配置管理
- 所有配置集中管理，使用dataclass
- 敏感配置从环境变量读取
- 提供配置验证函数

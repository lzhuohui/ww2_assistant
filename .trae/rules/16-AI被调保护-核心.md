# AI被调保护规范（核心）

## 规则9：被调用模块保护原则（核心规则）

### 核心约束
**在解决当前模块问题时，绝对禁止修改被调用模块**

### 模块分类定义
- **当前模块**：正在解决问题的模块（可修改）
- **被调用模块**：当前模块通过import调用的模块（禁止修改）
- **调用方模块**：调用当前模块的模块（禁止修改）

### 识别被调用模块的方法
在修改任何代码前，必须分析import语句：
```python
# 当前文件：module_a.py
from module_b import func_b  # ← module_b是被调用模块，禁止修改！
from module_c import ClassC   # ← module_c是被调用模块，禁止修改！
```

### 发现被调用模块有问题时的处理流程

**核心原则**：**先最大限度适配，后考虑克隆**

```
第1优先级 ⭐⭐⭐ 当前模块适配（首选）
第2优先级 ⭐⭐   包装器模式（备选）
第3优先级 ⭐    继承扩展（备选）
第4优先级      克隆调用（最后手段）
```

**重要区分**：
- **接口不匹配** → 适配即可，不需要克隆
- **功能不足** → 包装或继承即可，不需要克隆
- **内部逻辑bug** → 最后手段，才考虑克隆

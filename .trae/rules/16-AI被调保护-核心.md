# AI被调保护规范（核心）

## 规则9：被调用模块保护原则（核心规则）

### 核心约束
**在解决当前模块问题时，绝对禁止修改被调用模块**

### 模块分类定义
- **当前模块**：正在解决问题的模块（可修改）
- **被调用模块**：当前模块通过import调用的模块（禁止修改）
- **调用方模块**：调用当前模块的模块（禁止修改）

### 识别被调用模块的方法
在修改任何代码前，必须分析import语句：
```python
# 当前文件：module_a.py
from module_b import func_b  # ← module_b是被调用模块，禁止修改！
from module_c import ClassC   # ← module_c是被调用模块，禁止修改！
```

### 发现被调用模块有问题时的处理流程

**核心原则**：**先最大限度适配，后考虑克隆**

```
第1优先级 ⭐⭐⭐ 当前模块适配（首选）
第2优先级 ⭐⭐   包装器模式（备选）
第3优先级 ⭐    继承扩展（备选）
第4优先级      克隆调用（最后手段）
```

**重要区分**：
- **接口不匹配** → 适配即可，不需要克隆
- **功能不足** → 包装或继承即可，不需要克隆
- **内部逻辑bug** → 最后手段，才考虑克隆

### 强制要求（必须遵守）

#### 禁止事项
1. **禁止修改被调用模块的代码**
   - 不修改被调用模块的函数实现
   - 不修改被调用模块的类定义
   - 不修改被调用模块的变量定义

2. **禁止覆盖被调用模块的变量**
   - 不硬编码默认值覆盖被调用模块的默认值
   - 不重新定义被调用模块已有的变量
   - 不修改被调用模块的配置参数

#### 必须事项
1. **必须让被调用模块使用自己的默认值**
   - 如果用户没有指定值，传递None或不传递
   - 让被调用模块使用自己定义的默认值

2. **必须在调用前转换数据格式**
   - 如果数据格式不匹配，在调用前转换
   - 不修改被调用模块来适配数据格式

3. **必须在调用后处理异常结果**
   - 如果被调用模块返回的结果不符合预期，在调用后处理
   - 不修改被调用模块来改变返回结果

### 具体操作指引

#### 如何判断是否修改了被调用模块
1. 检查import语句，列出所有被调用模块
2. 检查修改的代码是否在被调用模块中
3. 如果是，则违反了规则

#### 如何判断是否覆盖了被调用模块的变量
1. 检查被调用模块中的变量定义
2. 检查当前模块中是否有相同名称的变量
3. 检查当前模块是否硬编码了被调用模块的默认值
4. 如果是，则违反了规则

#### 如何避免硬编码
1. 不在调用方设置默认值
2. 如果用户没有指定值，传递None或不传递
3. 让被调用模块使用自己定义的默认值

### 违规后果
- 其他调用该模块的功能出错
- 测试通过的模块失效
- 进入死循环修改

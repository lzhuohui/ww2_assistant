# 异常处理规范

## 基本原则
- 所有外部调用必须try-catch
- 不得静默忽略异常（空except）
- 异常信息需记录到日志

## 重试机制
- 网络相关操作必须包含重试
- 使用指数退避策略
- 最大重试次数可配置

```python
def retry_operation(func, max_retries=3):
    for i in range(max_retries):
        try:
            return func()
        except Exception as e:
            if i == max_retries-1:
                raise
            time.sleep(2 ** i)
```

## 降级策略
- 关键操作失败应有降级方案
- 降级后需记录日志

## 日志要求
- 日志文件保存在`logs/`，按日期分割
- 格式：时间戳 - 级别 - 模块 - 详细信息
- 避免记录敏感信息
- 关键步骤有INFO日志，异常有ERROR日志

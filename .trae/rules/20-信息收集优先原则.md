# 信息收集优先原则

## 规则概述

在处理各类问题前，先通过网络渠道系统性收集相关信息，并从成熟实例中提炼可复用的成熟逻辑。

## 适用场景判断

| 场景类型 | 适用程度 | 执行要求 |
|---------|---------|---------|
| 通用功能 | ⭐⭐⭐⭐⭐ | 必须先收集信息 |
| 最佳实践 | ⭐⭐⭐⭐⭐ | 必须先收集信息 |
| 特定业务 | ⭐⭐⭐ | 建议收集信息 |
| 创新功能 | ⭐⭐ | 可选收集信息 |
| 紧急修复 | ⭐ | 可跳过收集 |

## 执行标准

### 1. 搜索关键词
- 准确描述问题核心
- 使用中英文双语搜索
- 包含技术栈关键词

### 2. 信息来源优先级
1. 官方文档
2. 知名技术博客
3. 开源项目示例
4. 技术社区问答

### 3. 方案对比要求
- 至少对比2-3个方案
- 评估方案的优缺点
- 选择最适合当前场景的方案

## 执行流程

```
1. 问题分析 → 明确问题类型和需求
      ↓
2. 场景判断 → 判断是否需要收集信息
      ↓
3. 信息收集 → 执行网络搜索和筛选
      ↓
4. 方案评估 → 对比多个方案的优劣
      ↓
5. 方案选择 → 选择最适合当前场景的方案
      ↓
6. 实施验证 → 测试并验证效果
      ↓
7. 知识沉淀 → 记录有效方案供后续复用
```

## 知识沉淀要求

### 1. 记录内容
- 问题描述
- 收集到的方案
- 最终选择的方案及原因
- 实施效果

### 2. 记录位置
- 规则文件：将有效方案记录到 `.trae/rules/` 目录下的相关规则文件中
- 代码注释：在相关代码中添加参考链接
- 工具函数：将可复用代码封装为工具函数

## 灵活调整原则

### 1. 根据问题复杂度
- 简单问题：快速搜索即可
- 复杂问题：深入调研多个来源

### 2. 根据时间紧迫度
- 紧急情况：可跳过收集，直接处理
- 非紧急情况：完整执行收集流程

### 3. 根据信息可用性
- 有成熟方案：直接采用
- 无成熟方案：基于收集的信息自主设计

- [ ] 是否使用了准确的关键词搜索？
- [ ] 是否对比了多个方案？
- [ ] 是否选择了最适合的方案？
- [ ] 是否记录了有效方案？

## 案例参考

### 案例：表单验证功能
**问题描述**：需要实现账号输入格式验证功能

**收集过程**：
1. 搜索"表单验证最佳实践 用户体验"
2. 找到文章《实时验证的反模式与失败案例》
3. 提炼出验证时机、反馈方式等最佳实践

**最终方案**：
- 验证时机：失去焦点时验证
- 反馈方式：副标题显示提示信息
- 层级分明：区分严重错误和轻微提醒

**实施效果**：用户体验提升，错误率降低

## 注意事项

1. **避免过度搜索**：控制搜索时间，避免陷入信息过载
2. **保持独立思考**：参考成熟方案，但要根据实际需求调整
3. **验证方案可行性**：在采用前评估方案是否适合当前项目
4. **持续优化规则**：根据实际执行效果调整本规则

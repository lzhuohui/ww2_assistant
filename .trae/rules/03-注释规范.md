# 注释规范

## 注释原则
- 注释应解释"为什么"，而非"是什么"
- 代码应自解释，注释补充意图
- 更新代码时同步更新注释
- 注释使用中文

## 注释位置
- **注释放在代码行后面** - 简洁明了，不单独一行
- 示例：`DEFAULT_WIDTH = 120  # 下拉框默认宽度`

## 注释场景对照表

| 场景 | 是否需要注释 | 示例 |
|-----|------------|------|
| 用户指定变量 | ✅ 需要 | `DEFAULT_WIDTH = 120  # 下拉框默认宽度` |
| 类名 | ✅ 需要 | `class CustomDropDown:  # 自定义下拉框组件` |
| 变量名已清晰 | ❌ 不需要 | `self._selected_text = ft.Text(...)` |
| 方法名已清晰 | ❌ 不需要 | `def create(self):` |
| 标准方法 | ❌ 不需要 | `def __init__(self, ...):` |
| 规则条文 | ❌ 不在代码中 | 只在规范文件中 |

## 必须注释的场景
- 用户指定变量 - 说明用途
- 类名 - 说明类的用途
- 复杂算法逻辑
- 业务规则和约束
- 非常规实现原因
- API接口说明
- TODO/FIXME

## 禁止注释的场景
- 变量名已清晰的情况
- 方法名已清晰的情况
- 标准方法（如 `__init__`）
- 规则条文（只在规范文件中）

## 注释格式示例
```python
# *** 用户指定变量 - AI不得修改 ***
DEFAULT_WIDTH = 120        # 下拉框默认宽度
DEFAULT_HEIGHT = 32        # 下拉框默认高度
# *********************************

class CustomDropDown:  # 自定义下拉框组件
    
    def __init__(self, config: 界面配置, ...):
        self._selected_text = ft.Text(...)  # 无需注释，变量名已清晰
        self._dropdown_icon = ft.Icon(...)  # 无需注释，变量名已清晰
    
    def create(self):  # 无需注释，方法名已清晰
        return self._control
```

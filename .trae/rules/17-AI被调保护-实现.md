# AI被调保护规范（实现）

## 规则9：被调用模块保护原则（续）

#### 第1步：当前模块适配（首选）
在不修改被调用模块的情况下解决问题：
- 数据转换：在调用前转换数据格式
- 结果处理：在接收结果后处理异常
- 条件判断：添加前置条件检查

#### 第2步：包装器模式（备选）
通过包装器扩展功能，不修改原模块：
```python
class ModuleBAdapter:
    def __init__(self):
        self._module_b = module_b
    def enhanced_function(self):
        result = self._module_b.original_function()
        return self._process(result)
```

#### 第3步：继承扩展（备选）
通过继承扩展功能：
```python
class EnhancedClassB(module_b.ClassB):
    def new_method(self):
        pass
```

#### 第4步：模块克隆调用（最后手段）
**触发条件**：以上3种方案均无法解决，且必须修改被调用模块内部逻辑

**克隆流程**：
1. 克隆被调用模块：`@module-manager 克隆模块 [原模块] [新模块]`
2. 当前模块切换至新版
3. 测试验证
4. 实际运行测试其他调用方
5. 双版本并存决策
6. 长期维护与迁移

**双版本并存方案**：
- 当前模块使用新版（v2）
- 其他模块继续使用原版
- 未来按需逐一验证迁移

### 违规后果
修改被调用模块将导致：
- 其他调用该模块的功能出错
- 测试通过的模块失效
- 进入死循环修改（修复A→破坏B→修复B→破坏A）

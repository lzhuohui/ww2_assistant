# AI模块管理规范

## 规则6：不可变模块原则

### 核心约束
**一旦模块通过测试，即视为不可变（Immutable），禁止任何修改**

### 模块生命周期
```
[开发中] → [测试中] → [已冻结] → [已发布]
    ↑          ↑          ↑          ↑
  可修改    可修改     禁止修改   禁止修改
```

### 状态定义
- **开发中**：正在开发的模块，可以修改
- **测试中**：开发完成，正在测试，可以修复bug
- **已冻结**：测试通过，已冻结，禁止修改
- **已发布**：正式发布，禁止修改

### 状态转换控制

#### AI自动执行（无需确认）
- 开发中 → 测试中：AI完成开发后自动标记

#### 必须用户确认（关键决策点）
- 测试中 → 已冻结：**必须用户确认后才能冻结**
- 已冻结 → 开发中：**必须用户确认后才能解冻**
- 已冻结 → 已发布：**必须用户确认后才能发布**

### 禁止行为
- ❌ 修改已冻结/已发布的模块
- ❌ 以"优化"为由修改测试通过的模块
- ❌ 在解决A模块时修改B模块（即使B有问题）

### 发现问题时的正确处理
如果发现已冻结模块有问题：
1. **立即停止当前工作**
2. **向用户报告**：说明问题模块、问题描述、影响范围
3. **等待用户决策**：选项A（创建新任务）或选项B（授权解冻）
4. **不得擅自修改**

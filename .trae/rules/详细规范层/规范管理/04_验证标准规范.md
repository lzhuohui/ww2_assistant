# 规范管理 - 验证标准

## 五、验证标准

### 5.1 规则配置验证

| 验证项 | 通过标准 | 验证方法 |
|-------|---------|---------|
| 核心规范自动加载 | 新建对话自动显示规则引用 | 观察对话界面规则标识 |
| 详细规范智能匹配 | 相关任务自动加载对应规范 | 测试不同场景的规则加载情况 |
| 手动触发有效 | #Rule引用成功加载规则 | 执行手动引用测试 |
| 规范执行准确性 | 测试模块符合所有规范要求 | 代码审查验证 |
| 无规则冲突 | 规则间无矛盾或重复 | 规则内容对比检查 |

### 5.2 规范内容验证

| 验证项 | 通过标准 | 验证方法 |
|-------|---------|---------|
| 格式正确 | 符合YAML Front Matter规范 | 检查文件头格式 |
| 引用路径正确 | 所有#Rule引用路径有效 | 测试引用加载 |
| 内容完整 | 包含所有必需章节 | 内容审查 |
| 无冗余信息 | 无重复或无关内容 | 内容对比检查 |
| 描述准确 | description准确反映适用场景 | 场景匹配测试 |

### 5.3 执行效果验证

| 验证项 | 通过标准 | 验证方法 |
|-------|---------|---------|
| AI执行准确性 | 测试代码符合规范要求 | 代码审查验证 |
| 规则加载效率 | 规则加载时间合理 | 性能测试 |
| Token使用效率 | 规则内容占用token合理 | Token使用统计 |
| 用户体验 | 团队成员使用顺畅 | 问卷调查 |

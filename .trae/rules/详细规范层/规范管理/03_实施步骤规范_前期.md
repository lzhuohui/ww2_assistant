# 规范管理 - 实施步骤（前期）

## 四、实施步骤

### 阶段一：目录重构（1-2天）

1. **备份现有规则**
   ```bash
   cp -r .trae/rules .trae/rules_backup_$(date +%Y%m%d)
   ```

2. **创建新目录结构**
   ```bash
   mkdir -p .trae/rules/01_核心规范
   mkdir -p .trae/rules/02_详细规范
   mkdir -p .trae/rules/03_检查清单
   mkdir -p .trae/rules/04_测试用例
   ```

3. **迁移规范文件**
   - 将核心规范迁移到01_核心规范目录
   - 将详细规范迁移到02_详细规范目录
   - 将检查清单迁移到03_检查清单目录
   - 将测试用例迁移到04_测试用例目录

4. **删除冗余文件**
   - 删除重复的主文件（如代码结构规范.md）
   - 保留精简版作为核心规范

### 阶段二：规则配置（1天）

1. **配置核心规范**
   - 设置alwaysApply: true
   - 添加详细规范引用

2. **配置详细规范**
   - 设置alwaysApply: false
   - 优化description提高匹配精度

3. **配置检查清单**
   - 设置alwaysApply: false
   - 明确使用时机

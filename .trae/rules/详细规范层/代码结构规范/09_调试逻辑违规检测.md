# 调试逻辑违规自动检测

## 检测规则

AI生成或修改调试逻辑时，需执行检查：

### 1. 格式检查
- [ ] 使用`if __name__ == "__main__":`
- [ ] 使用`ft.run(lambda page: page.add(...))`
- [ ] 只更改了被测模块名称
- [ ] 其他代码与标准格式完全一致

### 2. 包装函数检查
- [ ] 有`def main(page):`等包装函数
- [ ] 有`def test_xxx():`等测试包装函数
- [ ] 有函数定义在`if __name__ == "__main__":`块内

### 3. 页面配置检查
- [ ] `page.title`
- [ ] `page.padding`
- [ ] `page.bgcolor`
- [ ] `page.xxx`属性

### 4. 参数检查
- [ ] 硬编码的非默认参数
- [ ] `title="自定义"`等字符串
- [ ] `param=value`等自定义值
- [ ] 非默认参数

### 5. 回调检查
- [ ] `def on_click(e):`等回调函数
- [ ] `def callback(e):`等回调函数
- [ ] `on_click=xxx`等回调参数
- [ ] 有回调函数包装

## 标准格式模板

```python
# *** 调试逻辑 ***
if __name__ == "__main__":
    ft.run(lambda page: page.add(XXX.create()))
```

**唯一允许的变化**：将`XXX`替换为被测模块的类名

## 违规处理

### 轻微违规
- **表现**：格式不规范但不影响功能
- **处理**：警告并建议修正

### 中度违规
- **表现**：使用了包装函数或页面配置
- **处理**：提示修正并重新检查

### 严重违规
- **表现**：硬编码参数或回调函数
- **处理**：拒绝执行并要求重新实现

# 函数参数规范

## 参数默认值

必须有默认值：
```python
def create(
    title: str="测试标题",
    icon: str="HOME",
    enabled: bool=True,
) -> ft.Container:
```

### 核心原则

- 所有参数应有必要的默认数据
- **默认值必须能展示完整的功能效果**
- 测试时能看到效果，才能判定是否正确

### 不合适的默认值（禁止）

| 类型 | 违规示例 | 问题 | 正确示例 |
|------|----------|------|----------|
| None | `icon: str=None` | 不显示，无法判定 | `icon: str="SETTINGS"` |
| 空字符串 | `icon: str=""` | 不显示，用户以为代码错误 | `icon: str="SETTINGS"` |
| 空列表 | `items: list=[]` | 不显示，无法验证功能 | `items: list=["选项A", "选项B"]` |
| 空字典 | `config: dict={}` | 不显示，无法验证功能 | `config: dict={"key": "value"}` |

### 为什么禁止空值默认值？

1. **测试看不到效果**：调试时无法验证功能是否正常
2. **用户困惑**：用户可能以为是"代码错误"
3. **无法展示功能**：无法展示模块的完整功能效果
4. **违反如实测试原则**：调试逻辑无法如实反映模块运行情况

**关联规范**：参见 #Rule:代码结构规范/07_调试逻辑规范

### 默认值来源规则

| 来源 | 适用场景 | 示例 |
|------|----------|------|
|直接硬编码|通用默认值，用户未指定| `title: str="测试标题"` |
|用户指定变量|用户明确指定的配置| `size: int=USER_TITLE_SIZE` |
|配置文件|通用配置数据| `配置.获取尺寸("字体","font_size_lg")` |

### 严格禁止

1. 禁止将默认值放在用户指定变量区域（用户明确指定除外）
2. 禁止使用DEFAULT_前缀的模块级变量作为函数默认值
3. 禁止在用户未指定时创建USER_变量
4. **禁止使用None、空字符串、空列表、空字典作为默认值**（除非模块设计明确需要"无值"状态）

## 参数传递格式（PEP 8）

函数定义调用时，`=`两边不加空格：
```python
def create(title: str="默认值", size: int=16):
    pass

ContainerTitle.create(title=title, text_size=USER_TITLE_SIZE, padding=3)
```

## 参数命名规范

- 使用snake_case命名
- 参数名称应有描述性
- 所有参数应有类型注解
- 返回值应有类型注解

```python
from typing import Callable, Optional

def create(
    title: str="默认值",
    on_click: Optional[Callable]=None,
) -> ft.Container:
    pass
```

## AI执行检查清单

在定义函数参数时，AI必须检查：

- [ ] 所有参数是否有默认值？
- [ ] 默认值是否能展示完整功能？
- [ ] 是否避免了None/空字符串/空列表/空字典？
- [ ] `=`两边是否没有空格？
- [ ] 是否有类型注解？

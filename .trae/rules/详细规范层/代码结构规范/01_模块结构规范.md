# 模块结构规范

## 标准模块结构

新建模块必须包含以下6个部分，顺序不得更改：

### 1. 文件头
```python
# -*- coding: utf-8 -*-
```

### 2. 文档字符串
```python
"""
模块名称：XXX
设计思路及联动逻辑:
    核心设计理念。
    1. 第一步
    2. 第二步
模块隔离原则:
    1. 不直接创建被调用模块的内容
    2. 不覆盖被调用模块的计算结果
    3. 用户指定变量除外
"""
```

### 3. 导入部分
- 标准库导入
- 第三方库导入
- 项目内部模块导入
- 不导入未使用的模块

### 4. 用户指定变量区域

#### 有用户指定变量时
```python
# *** 用户指定变量 - AI不得修改, 变量值必须生效 ***
USER_XXX = 16
# *********************************
```

#### 无用户指定变量时
```python
# *** 用户指定变量 - AI不得修改, 变量值必须生效 ***
# （用户未指定变量）
# *********************************
```

### 5. 类/函数定义
- 核心功能实现
- 明确输入输出类型
- 函数长度≤20行

### 6. 调试逻辑
```python
# *** 调试逻辑 ***
if __name__ == "__main__":
    ft.run(lambda page: page.add(XXX.create()))
```

## 注意事项
- 顺序不得更改
- 各部分之间空一行分隔
- 调试逻辑只能更改被测模块名称

# 开发流程 - 模块复用优先原则（修正代码）

### 2.2 修正代码时

**执行流程：**
```
1. 分析问题 → 2. 搜索相关模块 → 3. 评估是否可用现有模块替换 → 4. 执行修正
```

**检查要点：**
- 问题是否因为未使用现有模块导致？
- 是否可以用现有模块替换问题代码？
- 修正时是否引入了可复用的现有模块？

**示例场景：**
```python
# ❌ 错误：直接创建Text控件
ft.Text("标题", size=24, weight=ft.FontWeight.BOLD)

# ✅ 正确：使用现有的文本标签模块
LabelText.create(text="标题", role="h2")
```

# 开发流程 - 代码审查（基础流程）

> **⚠️ 代码审查必须包含模块复用检查**  
> 参考：#Rule:开发流程_模块复用优先

---

## 审查流程

### 步骤1：模块复用审查（新增，强制）

**任何代码审查的第一步，必须检查模块复用情况：**

#### 1.1 检查是否搜索现有模块
- [ ] 代码中是否使用了项目内现有模块？
- [ ] 如果未使用，是否说明了理由？
- [ ] 是否优先复用了单元模块？
- [ ] 是否优先复用了组件模块？

#### 1.2 检查是否存在重复功能
- [ ] 新代码功能是否与现有模块重复？
- [ ] 是否可以通过参数调整现有模块实现？
- [ ] 是否可以通过组合现有模块实现？

#### 1.3 检查是否正确复用
- [ ] 是否通过公开接口调用？
- [ ] 是否通过参数传递调整行为？
- [ ] 是否遵守模块隔离原则？

**审查结果记录：**
```
模块复用审查结果：
- 已复用模块：_______________________________
- 未复用但应复用的模块：_____________________
- 建议：_____________________________________
```

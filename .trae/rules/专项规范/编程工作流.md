# 编程工作流详细规程

## 一、各阶段详细要求

### 阶段1：需求澄清

| 项目 | 内容 |
|------|------|
| 目标 | 确保需求明确、完整、可执行 |
| 输出 | 需求确认清单、关键约束、预期交付物 |
| 验证 | 用户确认需求理解正确 |

**输出格式**：
```
【需求澄清】
- 功能需求：xxx
- 技术约束：xxx
- 预期交付：xxx
请确认需求理解是否正确？
```

### 阶段2：收集最佳实践

| 项目 | 内容 |
|------|------|
| 目标 | 收集技术方案参考 |
| 输出 | 2-3条核心最佳实践 |
| 验证 | 收集到有效参考 |

**输出格式**：
```
【最佳实践】
1. xxx
2. xxx
3. xxx
```

### 阶段3：提供三方案

| 项目 | 内容 |
|------|------|
| 目标 | 提供差异化方案供用户选择 |
| 输出 | 三个方案（简洁/标准/完整） |
| 验证 | 用户明确选择一个方案 |

**输出格式**：
```
【三方案】
方案A（简洁）：xxx
方案B（标准）：xxx
方案C（完整）：xxx
请选择方案？
```

### 阶段4：按规则执行

| 项目 | 内容 |
|------|------|
| 目标 | 按选定方案实现代码 |
| 输出 | 符合规范的代码 |
| 验证 | 代码实现完成 |

**执行流程**：
1. 调用《模块架构规范.md》判断模块类型
2. 调用《模块代码结构.md》中对应模板
3. 遵循《代码风格规范.md》
4. 遵循《安全规范.md》

### 阶段5：核查交付

| 项目 | 内容 |
|------|------|
| 目标 | 确保交付质量 |
| 输出 | 核查清单、交付内容 |
| 验证 | 核查清单全部通过 |

**执行流程**：
1. 调用《成果提交规范.md》执行测试验证
2. AI报告 + 用户确认
3. 核查清单确认

**输出格式**：
```
【核查清单】
- [ ] 代码符合模块结构规范
- [ ] 代码符合代码风格规范
- [ ] 测试逻辑正确
- [ ] 用户指定变量受保护
请确认交付内容？
```

## 二、用户交互要求

| 阶段 | 必须确认 |
|------|----------|
| 阶段1 | 需求澄清清单 |
| 阶段3 | 最终选择的方案 |
| 阶段4 | 涉及模块修改时 |
| 阶段5 | 交付内容确认 |

## 三、执行约束

每次阶段转换时，AI必须：
1. 告知用户当前进入的阶段
2. 说明该阶段的目标
3. 等待用户确认后再进入下一阶段

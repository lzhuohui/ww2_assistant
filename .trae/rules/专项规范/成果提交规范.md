# 成果提交规范

## 一、AI执行要求（强制）

AI在提交代码前必须执行测试并验证通过，确保交付质量。

### 1.1 执行流程

```
处理代码 → 运行测试逻辑 → AI验证 → 通过 → AI报告 → 用户确认
                                ↓
                              失败 → 修复 → 重新测试
```

### 1.2 必须执行

| 步骤 | 要求 |
|------|------|
| 1. 运行测试 | 执行 if __name__ == "__main__" 中的测试逻辑 |
| 2. AI验证 | 检查返回值类型、无异常 |
| 3. AI报告 | 说明变更内容+测试结果，请用户确认 |

### 1.3 禁止行为

| 禁止项 | 说明 |
|--------|------|
| 未经测试提交 | 禁止跳过测试直接提交代码 |
| 忽略测试错误 | 禁止忽略测试中的错误或异常 |
| 提交不可运行代码 | 禁止提交无法执行的代码 |

## 二、模块验证与提交标准

### 2.1 V2前端模块

**AI验证标准**：
- 运行无语法错误
- 运行无运行时异常
- 界面启动成功
- 销毁方法可正常调用

**AI提交报告格式**：
```
【V2前端模块交付】
- 模块层级：[说明模块所在层级：主入口/功能界面/卡片组/复合模块/基础模块]
- 变更内容：[说明修改了什么]
- 销毁验证：[说明销毁方法是否正常]
请运行代码确认界面效果是否符合预期？
```

### 2.2 V1脚本界面类

**AI验证标准**：
- 运行无语法错误
- 运行无运行时异常
- 界面启动成功

**AI提交报告格式**：
```
【界面类交付】
- 变更内容：[说明修改了什么]
- 界面功能：[说明界面功能]
请运行代码确认界面效果是否符合预期？
```

### 2.3 游戏动作类

**AI验证标准**：
- 返回值为True（布尔类型）
- 无异常抛出

**AI提交报告格式**：
```
【动作类交付】
- 测试动作：[说明测试的动作内容]
- 测试结果：返回True
请在游戏中确认动作效果是否正确？
```

### 2.4 游戏查询类

**AI验证标准**：
- 返回结果不为None
- 返回结果类型正确（字典/元组）
- 无异常抛出

**AI提交报告格式**：
```
【查询类交付】
- 查询目标：[说明查询什么]
- 查询结果：[打印查询结果]
请确认查询结果是否准确？
```

### 2.5 数据操作类

**AI验证标准**：
- 保存函数返回True
- 读取函数返回非None数据
- 保存后读取数据一致

**AI提交报告格式**：
```
【数据类交付】
- 操作内容：[说明保存/读取什么]
- 测试结果：数据一致性验证通过
请确认数据操作是否正确？
```

## 三、提交结果判定

### 3.1 通过条件

| 条件 | 说明 |
|------|------|
| 无语法错误 | 代码可正常解析 |
| 无运行时错误 | 测试逻辑可正常执行 |
| 返回值正确 | 函数返回预期类型和值 |

### 3.2 失败处理

```
测试失败 → 分析错误原因 → 修复代码 → 重新测试 → 直至通过
```

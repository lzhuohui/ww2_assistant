# 代码风格规范

## 一、文件格式

### 1.1 文件编码
所有Python文件必须使用UTF-8编码：
```python
# -*- coding: utf-8 -*-
```

### 1.2 文件结构
Python文件结构根据模块类型有所不同：

**界面类（6区域）**：
1. 文件开头区域（编码声明）
2. 设计记录区域（模块文档字符串）
3. 模块导入区域
4. 指定变量区域（USER_变量为模块组成部分）
5. 类/函数区域
6. 测试逻辑区域

**其他类（5区域）**：
1. 文件开头区域（编码声明）
2. 设计记录区域（模块文档字符串）
3. 模块导入区域
4. 类/函数区域
5. 测试逻辑区域（含测试数据）

## 二、命名规范

### 2.1 变量命名

| 变量类型 | 命名规范 | 示例 |
|----------|----------|------|
| 界面类指定变量 | USER_前缀，大写字母，下划线分隔 | `USER_WINDOW_TITLE` |
| 其他类测试数据 | USER_前缀，大写字母，下划线分隔 | `USER_TEST_X` |
| 模块级常量 | 大写字母，下划线分隔 | `MAX_RETRY_COUNT` |
| 函数局部变量 | 小写字母，下划线分隔 | `user_input`, `file_path` |
| 函数参数 | 小写字母，下划线分隔 | `page`, `data_file` |

**变量位置区别**：
- 界面类：USER_变量在模块级别（独立区域），为模块组成部分
- 其他类：USER_变量在if __name__内部，为测试数据

### 2.2 函数命名

| 函数类型 | 命名规范 | 示例 |
|----------|----------|------|
| 主函数 | 小写字母，下划线分隔 | `main`, `execute_action` |
| 辅助函数 | 动词+名词，小写下划线 | `save_data`, `load_config` |
| 查询函数 | 动词+名词，小写下划线 | `find_element`, `check_status` |

### 2.3 类命名
使用大驼峰命名法：
```python
class GameController:
    pass

class DataManager:
    pass
```

### 2.4 目录/文件命名

| 类型 | 语言 | 示例 |
|------|------|------|
| 目录/文件 | 中文 | `专项规范/`、`编程工作流.md` |
| 类/函数/变量 | 英文 | `UserManager`、`create_user()` |

**目的**：兼顾阅读方便，转APK前批量改目录/文件名为英文。

## 三、注释规范

### 3.1 模块文档字符串
每个模块必须有设计记录区域：
```python
"""
模块名称：[文件名称.py]  # 如：click_action.py
模块功能：[功能描述]
实现步骤：
- [步骤1]
- [步骤2]
- [步骤3]
"""
```

### 3.2 函数文档字符串
重要函数必须有文档字符串：
```python
def save_data(data):
    """保存数据到文件"""
    # 实现代码
```

### 3.3 行内注释
用户指定变量必须有详细注释：
```python
# *** 用户指定变量: 变量值必须生效,AI不得更改数据 ***
USER_WINDOW_TITLE = "游戏辅助工具"  # 主窗口标题，显示在窗口顶部
# *********************************
```

## 四、代码格式

### 4.1 缩进
使用4个空格缩进，禁止使用Tab。

### 4.2 行长度
每行不超过88个字符。

### 4.3 空行
- 导入区域后空一行
- 指定变量区域后空一行
- 函数之间空一行
- 类方法之间空一行

## 五、导入规范

### 5.1 导入顺序
1. 标准库导入
2. 第三方库导入
3. 项目内部模块导入

```python
# 标准库
import json
import os
import time

# 第三方库
import flet as ft
import pyautogui

# 项目内部模块
# from utils import helper
```

### 5.2 导入格式
- 每个导入独占一行
- 禁止使用通配符导入
- 使用别名时保持一致性

## 六、函数设计

### 6.1 函数长度
函数长度建议不超过20行，复杂逻辑应拆分为多个小函数。

### 6.2 参数数量
函数参数数量建议不超过5个，过多参数应考虑使用字典或对象封装。

### 6.3 参数默认值
所有参数必须有默认值，默认值必须能展示完整功能效果：

```python
def create(
    title: str = "默认标题",
    icon: str = "SETTINGS",
    enabled: bool = True,
) -> ft.Container:
```

### 6.4 参数注释
复杂/非显而易见参数可添加注释，注释在参数后同一行：

```python
def create(
    title: str = "默认标题",      # 显示标题
    icon: str = "SETTINGS",       # 图标名称
) -> ft.Container:
```

**禁止使用docstring Args**：类型注解已提供足够信息，无需重复。

## 七、冗余代码禁止

| 类型 | 说明 |
|------|------|
| 重复代码 | 相同逻辑应抽取为函数 |
| 死代码 | 删除未使用的变量/函数/导入 |
| 注释代码 | 删除被注释的代码块 |

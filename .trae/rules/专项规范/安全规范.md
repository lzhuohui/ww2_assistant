# 安全规范

## 一、密钥与敏感信息

### 1.1 绝对禁止

| 禁止项 | 说明 |
|--------|------|
| 硬编码密钥 | 禁止在代码中直接写入密钥、密码、Token |
| 提交密钥 | 禁止将密钥提交到版本控制系统 |
| 日志密钥 | 禁止在日志中输出敏感信息 |

### 1.2 密钥管理要求

- 密钥必须存储在**环境变量**或**密钥管理服务**中
- 使用 `.env` 文件时，必须将其加入 `.gitignore`
- 敏感配置应通过**配置中心**或**密钥管理服务**获取

## 二、输入校验

### 2.1 数据校验要求

| 场景 | 要求 |
|------|------|
| 用户输入 | **必须**进行校验和清理 |
| 文件上传 | **必须**校验文件类型和大小 |
| API参数 | **必须**校验参数类型和范围 |

### 2.2 校验原则

- **最小权限**：只接受最小必要的数据
- **白名单优先**：优先使用白名单校验
- **防御性编程**：假设所有输入都是恶意的

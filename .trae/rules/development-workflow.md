# 开发流程规范

## 新建模块流程

1. **搜索复用**：先搜索项目中是否有可复用的模块
2. **参考结构**：参考现有模块的代码结构
3. **遵循规范**：严格按照代码结构规范创建模块

## 模块复用优先

### 搜索方法
- 使用 SearchCodebase 工具搜索功能关键词
- 使用 Grep 工具搜索类名、函数名
- 检查 前端/用户设置界面/组件模块 目录

### 复用判断标准
- 功能相似度 > 80% 可直接复用
- 功能相似度 50-80% 可扩展后复用
- 功能相似度 < 50% 新建模块

## 模块调用规范

### 前置检查
1. 确认被调用模块的公开接口
2. 确认参数类型和默认值
3. 从被调用模块导入默认值，禁止硬编码

### 调用原则
- 只通过公开接口访问模块
- 不修改被调用模块的内部实现
- 不覆盖被调用模块的用户指定变量

## 代码审查清单

- [ ] 模块结构完整（5部分）
- [ ] 文档字符串格式正确
- [ ] 用户指定变量区域存在
- [ ] 导入顺序正确
- [ ] 函数参数有默认值
- [ ] 调试逻辑格式正确

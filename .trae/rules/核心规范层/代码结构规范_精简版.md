# 代码结构规范（精简版）

## 核心原则
- **模块完整性**：包含文件头、导入、用户变量、类/函数、调试逻辑
- **用户变量保护**：AI不得修改用户变量区域
- **如实测试**：调试逻辑如实反映模块运行情况
- **模块隔离**：调用者不得修改被调模块实现
- **无冗余**：移除未使用的变量、导入、重复代码

## 模块结构
```python
# -*- coding: utf-8 -*-
"""模块名称：XXX | 设计思路 | 隔离原则"""
import flet as ft

# *** 用户指定变量 - AI不得修改 ***
# （用户未指定变量）
# *********************************

class XXX:
    @staticmethod
    def create(param1: str="默认值", param2: int=16) -> ft.Container:
        return ft.Container()

if __name__ == "__main__":
    ft.run(lambda page: page.add(XXX.create()))
```

## 关键规范
1. **文档字符串**：模块级必须包含（名称、思路、隔离原则），禁止包含层级说明
2. **用户变量区域**：区域必须存在，未指定时标注 `# （用户未指定变量）`；有变量时必须添加中文注释
3. **导入顺序**：标准库 → 第三方库 → 项目模块
4. **函数参数**：所有参数应有默认值，`=`两边不加空格
5. **模块隔离**：只通过公开接口访问模块
6. **调试逻辑**：使用统一结构，禁止添加初始化代码
7. **代码质量**：函数≤20行，类>300行考虑拆分
8. **禁止冗余别名**：禁止创建或导出中文别名

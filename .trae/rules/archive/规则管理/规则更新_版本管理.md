# 规则更新 - 版本管理

## 三、版本管理

### 3.1 版本号规范

```
v主版本号.次版本号.修订号
```

#### 主版本号

**更新条件**：重大变更，不兼容的API修改

**示例**：v1.0.0 → v2.0.0：模块结构标准重大变更

#### 次版本号

**更新条件**：功能新增，向下兼容的功能性新增

**示例**：v1.0.0 → v1.1.0：新增技能使用规范

#### 修订号

**更新条件**：bug修复，向下兼容的问题修正

**示例**：v1.0.0 → v1.0.1：修正引用路径错误

### 3.2 版本记录

| 版本号 | 更新日期 | 更新内容 | 影响范围 | 变更类型 |
|-------|---------|---------|---------|---------|
| v1.0.0 | 2026-03-20 | 初始版本 | 全部规范 | 初始版本 |

### 3.3 版本控制

#### 控制工具

- **Git**：版本控制
- **Git Flow**：分支管理
- **Semantic Versioning**：版本号规范

### 3.4 版本发布

#### 发布流程

1. 更新版本号
2. 更新变更日志
3. 提交到Git
4. 打标签
5. 通知团队

#### 变更日志格式

```markdown
## v1.0.0 (2026-03-20)

### 新增功能
- 新增技能使用规范
- 新增规则动态更新机制

### 修复问题
- 修正引用路径错误
- 优化规则描述

### 已知问题
- 无

### 升级指南
1. 更新.trae/rules目录下的规则文件
2. 新开对话以使用新规则
3. 参考技能使用规范使用新功能
```

# AI执行标准 - 核心原则

## 模块结构（5部分）

```python
# -*- coding: utf-8 -*-
"""模块名称：XXX | 设计思路 | 模块隔离原则"""
import flet as ft

# *** 用户指定变量 - AI不得修改 ***
# （用户未指定变量）
# *********************************

class XXX:
    @staticmethod
    def create(...) -> ft.Container:
        return ft.Container()

if __name__ == "__main__":
    ft.run(lambda page: page.add(XXX.create()))
```

## 核心原则

### 用户变量保护
- **区域必须存在**：每个模块必须包含用户指定变量区域
- **禁止修改**：AI不得修改用户指定变量区域的任何内容
- **优先级**：函数参数 > 用户指定变量 > 配置文件 > 代码默认值

### 模块隔离
- **禁止往前覆盖**：不修改被调用方的用户指定变量
- **通过参数传递**：调用方通过函数参数传递配置
- **公开接口访问**：只通过公开接口访问模块

### 如实测试
- **标准格式**：`ft.run(lambda page: page.add(XXX.create()))`
- **禁止添加**：初始化代码、页面属性设置、非默认参数

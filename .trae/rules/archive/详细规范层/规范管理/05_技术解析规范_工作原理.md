# 规范管理 - 技术解析（工作原理）

## 一、Trae IDE规则处理机制

### 1.1 规则处理工作原理

```
用户输入 → 规则匹配引擎 → 上下文注入 → AI处理 → 输出生成
                ↓
        [alwaysApply规则自动加载]
        [globs匹配规则条件加载]
        [description智能匹配]
        [#Rule手动触发加载]
```

**关键技术点：**
1. **上下文窗口管理**：规则内容占用token，影响AI可用上下文
2. **规则优先级**：手动触发 > 始终生效 > 智能匹配 > 文件匹配
3. **冲突解决**：后加载规则覆盖先加载规则（同优先级）

### 1.2 规范解析流程

1. **预处理阶段**：解析YAML Front Matter，确定规则生效范围
2. **匹配阶段**：检查globs匹配、分析description相关性、检测#Rule引用
3. **注入阶段**：将匹配的规则内容注入系统提示，建立规则间引用关系
4. **执行阶段**：AI基于注入的规范生成响应，实时应用规范约束

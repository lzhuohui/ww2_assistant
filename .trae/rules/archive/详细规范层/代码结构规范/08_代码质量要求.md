# 代码质量要求

## 无冗余

- 移除未使用的变量
- 移除未使用的导入
- 移除重复的代码
- 移除重复的注释

## 格式正确

- 符合PEP 8规范
- 函数参数传递时`=`两边不加空格
- 文档字符串中的描述性文本`=`两边加空格

```python
# 正确
def create(title: str = "默认值") -> ft.Container:
    ContainerTitle.create(title=title, padding=3)
```

## 运行效率

- 无运行时错误
- 无未定义的变量
- 无死循环
- 合理的算法复杂度

## 可读性

- 代码结构清晰
- 注释简洁明了
- 变量命名规范
- 函数长度≤20行（仅计算代码行，不含注释和空行）
- 类超过300行考虑拆分

### 函数长度计算说明

    if config is None:                                 # 第2行：代码

    # 获取主题颜色                                    # 注释行，不计入

## 代码设计规范

### 函数
- 只做一件事
- 长度≤20行
- 避免多层嵌套
- 明确输入输出
- 无副作用

### 类
- 相关代码集中
- 适当抽象层次
- 超过300行考虑拆分
- 单一职责
- 封装完整

### 模块划分
- 按功能垂直切分
- 避免循环依赖
- 接口清晰
- 高内聚
- 低耦合

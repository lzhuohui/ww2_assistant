# 导入规范

## 适用场景
- 添加新的导入语句
- 整理现有导入语句
- 审查导入语句合规性

## 导入顺序

必须按照以下顺序排列导入语句：

```python
# 1. 标准库导入
import os
import sys
from typing import Callable, Optional
# 2. 第三方库导入
import flet as ft
# 3. 项目内部模块导入
from 前端.用户设置界面.核心接口.主题提供者 import ThemeProvider
from 前端.用户设置界面.单元模块.XXX import XXX
```

## 导入原则

### 1. 分组原则
- 标准库导入在前
- 第三方库导入在中
- 项目内部模块导入在后

### 2. 排序原则
- 每组内按字母顺序排序
- `import`语句在前，`from...import`语句在后

### 3. 不导入未使用的模块
- 移除所有未使用的导入
- 定期检查并清理冗余导入

## 禁止事项
- ❌ 使用通配符导入（`from module import *`）
- ❌ 一行导入多个模块（`import os, sys`）
- ❌ 导入未使用的模块
- ❌ 循环导入

## 检查清单
- [ ] 导入顺序正确（标准库 → 第三方库 → 项目内部）
- [ ] 每组内按字母顺序排序
- [ ] 无未使用的导入
- [ ] 无通配符导入
- [ ] 无循环导入

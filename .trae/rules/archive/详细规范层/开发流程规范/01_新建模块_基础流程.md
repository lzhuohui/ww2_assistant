# 开发流程 - 新建模块（基础流程）

> **⚠️ 重要前提：新建模块前必须先执行【模块复用优先】原则**  
> 参考：#Rule:开发流程_模块复用优先

---

## 前置检查（强制）

### 步骤0：模块复用评估（必须先执行）

**任何新建模块操作前，必须回答以下问题：**

1. **是否已全局搜索现有模块？**
   - [ ] 已搜索单元模块目录
   - [ ] 已搜索组件模块目录
   - [ ] 已搜索工具模块目录
   - [ ] 已搜索核心接口目录

2. **是否存在功能匹配的模块？**
   - [ ] 功能完全匹配 → **停止新建，直接复用**
   - [ ] 功能部分匹配 → **评估参数调整可行性**
   - [ ] 无匹配模块 → **继续新建流程**

3. **是否可以通过组合现有模块实现？**
   - [ ] 可以组合 → **创建组合层，调用现有模块**
   - [ ] 无法组合 → **继续新建流程**

**决策结果记录：**
```
新建模块理由：_________________________________
已评估的现有模块：_____________________________
不可复用的原因：_______________________________
```

## 新建模块流程

### 步骤1：确定模块定位

**仅在步骤0确认需要新建模块后执行：**

- 确定模块层级（单元层/组件层/页面层）
- 确定模块职责（单一职责原则）
- 确定对外接口

### 步骤4：测试验证

- 使用调试逻辑验证功能正确性
- 确保模块可独立运行
- 验证模块隔离原则

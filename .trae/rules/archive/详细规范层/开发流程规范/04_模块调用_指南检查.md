# 开发流程 - 模块调用（指南和检查）

## 模块选择指南

| 功能需求 | 优先使用模块 | 避免直接使用 |
|----------|-------------|-------------|
| 文本显示 | `LabelText` | `ft.Text` |
| 按钮 | `Button` | `ft.ElevatedButton`/`ft.TextButton` |
| 容器 | `GenericContainer`/`卡片容器` | `ft.Container` |
| 输入框 | `输入框` | `ft.TextField` |
| 下拉框 | `下拉框` | `ft.Dropdown` |

## 模块层级选择

```
需求分析 → 单元模块？ → 组件模块？ → 组合模块
```

## 禁止行为

| 禁止项 | 说明 | 正确做法 |
|--------|------|----------|
| ❌ 直接使用flet基础控件 | 不经过封装直接使用ft.Text等 | 使用项目内的封装模块 |
| ❌ 绕过公开接口 | 直接修改模块内部实现 | 只通过create()等公开接口调用 |
| ❌ 访问私有属性 | 访问`_`前缀的属性和方法 | 只使用公开属性和方法 |
| ❌ 复制模块代码 | 复制现有模块代码到本地使用 | 导入并调用现有模块 |

## 执行检查清单

- [ ] 步骤0完成：已确认最佳模块选择
- [ ] 步骤1完成：导入路径正确，顺序规范
- [ ] 步骤2完成：通过公开接口调用，参数传递正确
- [ ] 步骤3完成：组合使用时复用现有模块
- [ ] 步骤4完成：正确获取结果，不违反隔离原则

## 执行口诀

> **"先选模块，再调接口；参数调整，组合复用；隔离原则，绝不违反"**

# 模块隔离规范 - 核心原则

## 1. 接口隔离原则
- **公开接口**：明确标记对外提供的方法和属性
- **私有实现**：使用`_`前缀标记内部方法和属性
- **调用限制**：调用方只能通过公开接口访问模块

## 2. 数据隔离原则
- **内部数据保护**：私有属性使用`_`前缀
- **返回副本**：公开方法返回数据副本，不暴露内部引用
- **禁止直接修改**：调用方不得直接修改被调模块的内部数据

## 3. 依赖隔离原则
- **依赖注入**：通过构造函数或方法参数注入依赖
- **接口依赖**：只依赖公开接口，不依赖实现细节
- **避免循环依赖**：模块间依赖关系应为有向无环图

## 4. 状态隔离原则
- **状态封装**：模块状态由模块自身管理
- **状态查询**：提供公开方法查询状态
- **状态修改**：通过公开方法修改状态

## 5. 模块修改原则
- **影响评估**：修改被调用模块前必须评估对其他调用者的影响
- **用户确认**：修改被调用模块必须获得用户的明确同意
- **向下兼容**：修改被调用模块时应保持向下兼容性
- **测试验证**：修改后必须进行完整测试，确保不影响其他调用者

## 6. 默认值传递原则

### 6.1 默认值类型区分
| 类型 | 前缀 | 定义位置 | 使用场景 |
|------|------|----------|----------|
| **用户指定变量** | `USER_` | 用户指定变量区域 | 用户明确指定的可调参数 |
| **模块内部默认值** | `DEFAULT_` | 模块级常量 | 模块内部使用的固定值 |

### 6.2 用户指定变量（优先）
- **定义位置**：用户指定变量区域（`# *** 用户指定变量 ***`）
- **命名规范**：`USER_`前缀（如`USER_WIDTH`）
- **使用方式**：函数参数默认值引用用户指定变量
- **示例**：
  ```python
  # 用户指定变量区域
  USER_WIDTH = 20  # 容器宽度
  
  def create(width: int = USER_WIDTH):  # 引用用户指定变量
      pass
  ```

### 6.3 模块内部默认值
- **使用场景**：模块内部计算使用的固定值，非用户可调
- **命名规范**：`DEFAULT_`前缀（如`DEFAULT_PADDING`）
- **示例**：
  ```python
  DEFAULT_PADDING = 10  # 内部使用的默认边距
  
  def calculate():
      padding = DEFAULT_PADDING  # 内部使用
  ```

### 6.4 获取默认值原则
- **用户指定变量**：从被调用模块导入`USER_`前缀变量
- **模块内部默认值**：从被调用模块导入`DEFAULT_`前缀常量
- **禁止硬编码**：调用者不得自行定义默认值
- **单一来源**：默认值只在定义处维护，确保一致性

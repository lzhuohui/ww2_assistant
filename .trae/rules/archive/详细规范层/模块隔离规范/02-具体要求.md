# 模块隔离规范 - 具体要求

## 1. 命名规范
- **公开成员**：使用正常命名（如`process_data`）
- **私有成员**：使用下划线前缀（如`_internal_cache`）
- **保护成员**：使用双下划线前缀（如`__private_method`）

## 2. 访问控制
- **禁止访问**：调用方不得访问以下内容：
  - 私有属性和方法（`_`前缀）
  - 保护属性和方法（`__`前缀）
  - 内部模块变量（非公开导出）

## 3. 数据处理
- **返回策略**：
  - 不可变数据：可以直接返回
  - 可变数据：返回副本或深拷贝
  - 集合类型：返回新的集合实例

## 4. 依赖管理
- **依赖声明**：明确声明模块的外部依赖
- **依赖注入**：优先使用依赖注入模式
- **依赖解耦**：通过接口抽象减少模块间耦合

## 5. 违反处理

### 5.1 轻微违规
- **表现**：访问非关键私有成员
- **处理**：警告并建议修正

### 5.2 中度违规
- **表现**：直接修改被调模块的内部状态
- **处理**：提示修正并重新检查

### 5.3 严重违规
- **表现**：破坏模块隔离性，修改核心私有数据
- **处理**：拒绝执行并要求重新实现

## 6. 示例

### 6.1 正确示例
```python
# ✅ 正确：通过公开接口访问
class UserService:
    def __init__(self):
        self._users = []  # 私有属性
    
    def get_user(self, user_id):
        # 公开方法
        return next((u for u in self._users if u.id == user_id), None)
```

### 6.2 错误示例
```python
# ❌ 错误：直接访问私有属性
service = UserService()
users = service._users  # 违反模块隔离原则
```

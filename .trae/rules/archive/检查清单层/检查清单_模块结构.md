# 检查清单 - 模块结构

## 模块结构检查

### 文件头
- [ ] 编码声明正确：`# -*- coding: utf-8 -*-`
- [ ] 位于文件第一行

### 文档字符串
- [ ] 包含模块名称（中文）
- [ ] 包含设计思路及联动逻辑
- [ ] 包含模块隔离原则
- [ ] 无冗余信息（无数据来源、使用场景等）

### 导入部分
- [ ] 导入顺序正确（标准库 → 第三方库 → 项目内部）
- [ ] 每组内按字母顺序排序
- [ ] 无未使用的导入
- [ ] 无通配符导入

### 用户指定变量区域
- [ ] 使用标准格式隔离区域
- [ ] 区域必须存在（不可省略）
- [ ] 用户意图判断正确：
  - 用户明确指定变量 → 定义具体变量
  - 用户未指定变量 → 标注 `# （用户未指定变量）`
- [ ] 变量使用`USER_`前缀（如有变量）
- [ ] 变量值在代码中实际生效（如有变量）
- [ ] 未被二次覆盖
- [ ] 未将函数默认参数放在此区域

### 类/函数定义
- [ ] 核心功能实现完整
- [ ] 明确输入输出类型
- [ ] 函数长度≤20行

### 调试逻辑
- [ ] 使用标准格式：`ft.run(lambda page: page.add(XXX.create()))`
- [ ] 无包装函数
- [ ] 无页面配置
- [ ] 无硬编码参数
- [ ] 无回调包装

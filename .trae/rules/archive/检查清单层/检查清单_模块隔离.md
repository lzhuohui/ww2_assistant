# 检查清单 - 模块隔离

## 模块隔离原则检查

### 调用者检查
- [ ] 只通过公开接口访问模块
- [ ] 避免访问私有属性（`_`前缀）
- [ ] 避免修改被调模块的内部数据
- [ ] 通过接口修改被调模块的状态
- [ ] 避免依赖被调模块的实现细节
- [ ] 避免直接修改被调模块的用户指定变量
- [ ] 通过参数传递覆盖被调模块的默认值
- [ ] 从被调用模块返回的容器中获取尺寸信息

### 被调模块检查
- [ ] 明确标记公开接口
- [ ] 将内部实现标记为私有
- [ ] 返回数据副本而非内部引用
- [ ] 提供了清晰的接口文档
- [ ] 隐藏了实现细节
- [ ] 正确设置用户指定变量区域
- [ ] 保护用户指定变量不被修改
- [ ] 内部处理所有必要的初始化

## 调试逻辑规范检查

### 格式检查（强制）
- [ ] 使用标准格式：`if __name__ == "__main__":`
- [ ] 使用lambda表达式：`ft.run(lambda page: page.add(...))`
- [ ] 只能更改被测模块名称
- [ ] 其他代码必须与标准格式完全一致

### 严格禁止项
- [ ] **禁止**使用 `def main(page):` 包装函数
- [ ] **禁止**设置 `page.title`, `page.padding` 等页面属性
- [ ] **禁止**传入硬编码参数（使用模块默认值）
- [ ] **禁止**创建额外的回调函数
- [ ] **禁止**添加额外的初始化代码
- [ ] **禁止**修改被测模块的参数

### 功能检查
- [ ] 使用真实配置和参数
- [ ] 测试所有公开接口
- [ ] 无额外的干扰代码
- [ ] 测试结果可复现

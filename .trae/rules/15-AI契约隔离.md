# AI契约隔离规范

## 规则7：契约先行原则（Design by Contract）

### 核心思想
**先定义接口契约，后实现代码**

### 契约内容
每个模块必须定义以下契约：
1. **函数签名**：输入参数、返回值、异常
2. **前置条件**：调用前必须满足的条件
3. **后置条件**：调用后保证的结果
4. **性能约束**：时间复杂度、空间复杂度

### 开发流程
```
定义契约 → 用户审查 → 契约锁定 → 实现代码 → 契约验证 → 模块冻结
```

### 关键约束
- 契约一旦锁定，不得修改
- 实现必须严格符合契约
- 契约是模块间的法律

## 规则8：上下文隔离原则

### 核心思想
**AI只能看到允许修改的模块，看不到已冻结的模块**

### 上下文分层
- **全局上下文**：架构图、依赖关系、契约汇总
- **任务上下文**：当前任务、允许修改的模块清单
- **执行上下文**：当前文件、相关代码片段

### 关键约束
- 已冻结模块的代码对AI不可见（只能看到接口）
- AI只能修改白名单中的模块
- 禁止修改的模块代码从上下文中移除

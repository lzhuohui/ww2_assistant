## 二、核心规则速查

### 2.1 模块结构（5部分）

```python
# 1. 文件头
# -*- coding: utf-8 -*-
"""模块名称：XXX | 设计思路 | 模块隔离原则"""

# 2. 导入
import flet as ft

# 3. 用户指定变量
# *** 用户指定变量 - AI不得修改 ***
# （用户未指定变量）
# *********************************

# 4. 类/函数定义
class XXX:
    @staticmethod
    def create(...) -> ft.Container:
        return ft.Container()

# 5. 调试逻辑
if __name__ == "__main__":
    ft.run(lambda page: page.add(XXX.create()))
```

### 2.2 配置优先级

```
函数参数 > 用户指定变量 > 配置文件 > 代码默认值
```

### 2.3 模块隔离原则

- **禁止往前覆盖**：不修改被调用方的用户指定变量
- **通过参数传递**：调用方通过函数参数传递配置
- **公开接口访问**：只通过公开接口访问模块

## 五、Trae IDE 规则机制说明

### 5.1 记忆机制

Trae IDE 通过以下方式实现规则记忆：
- **alwaysApply**: true 的规则自动加载到上下文
- **description**: 用于智能匹配用户意图
- **globs**: 指定规则适用的文件类型
- **#Rule引用**: 建立规则间关联

### 5.2 字符限制

- 单条文内容 ≤ 1000 字符
- 超限文件需拆分或精简
- 保持核心信息完整性

### 5.3 最佳实践

1. **核心规范精简**：核心规则保持简洁，详细内容放在详细规范
2. **合理引用**：使用 #Rule 引用避免重复
3. **分层管理**：按功能模块、执行优先级分层
4. **定期审查**：定期检查规则间冲突和重复

# 代码结构规范

## 模块代码结构

每个模块必须遵循以下结构：

```python
# -*- coding: utf-8 -*-
"""
模块名称 - 层级

设计思路：
- 设计原理说明
- 关键决策点

功能：
- 功能列表
"""

import ...

# *** 用户指定变量 - AI不得修改 ***
# 规则：
# 1. 用户指定变量必须加载，不得在同一模块内再次覆盖
# 2. 调用模块可以覆盖被调用模块的默认值（优先级：调用方 > 被调用方 > 配置文件）
# 3. 除指定变量数据外，其他数据一律来自配置文件或被调用模块
# (用户指定的变量放在这里，用户没有指定之前就空着)
DEFAULT_WIDTH = 120        # 默认宽度
# *********************************

class ClassName:  # 类说明
    ...

# ==================== 调试逻辑 ====================
if __name__ == "__main__":
    # 1. 界面配置初始化
    配置 = 界面配置()
    
    # 2. 自动加载用户数据覆盖默认值（在界面配置.__init__中自动完成）
    
    # 3. 正常启动被测模块
    def main(page: ft.Page):
        page.padding = 0
        page.bgcolor = 配置.当前主题颜色["bg_primary"]
        page.add(被测模块(配置).render())  # 只能更改此处**被测调用模块名称**
    
    ft.run(main)
```

## 调试逻辑规范

### 核心原则
- **只能更改被测调用模块名称**，杜绝任何硬编码
- **其他一律不得修改**，确保最大程度如实测试真实的模块运行状态

### 调试逻辑结构

```python
# ==================== 调试逻辑 ====================
if __name__ == "__main__":
    # 1. 界面配置初始化
    配置 = 界面配置()
    
    # 2. 自动加载用户数据覆盖默认值（在界面配置.__init__中自动完成）
    
    # 3. 正常启动被测模块
    def main(page: ft.Page):
        page.padding = 0
        page.bgcolor = 配置.当前主题颜色["bg_primary"]
        page.add(被测模块(配置).render())  # 只能更改此处**被测调用模块名称**
    
    ft.run(main)
```

### 调试逻辑规则

| 规则 | 说明 |
|------|------|
| 禁止硬编码 | 所有配置必须从配置对象获取 |
| 禁止包装 | 不添加任何额外的Container、padding等 |
| 禁止边界测试 | 边界测试应在单元测试中进行，不在调试逻辑中 |
| 统一结构 | 所有模块使用相同的调试逻辑结构 |
| 只改模块名 | 只能更改被测调用模块名称，其他一律不得修改 |

### 调试逻辑目的
1. **如实测试**：最大程度如实测试真实的模块运行状态
2. **标准化**：所有模块使用相同的调试逻辑结构
3. **易于维护**：调试逻辑统一，减少出错可能
4. **避免干扰**：不添加任何额外逻辑，避免干扰模块测试

## 模块文档字符串规范

每个模块必须在开头包含文档字符串，说明：

| 内容 | 必需 | 说明 |
|-----|------|------|
| 模块名称 - 层级 | ✅ | 如"自定义下拉框 - 组件层" |
| 设计思路 | ✅ | 设计原理、关键决策点 |
| 功能 | ✅ | 功能列表 |

**示例：**
```python
"""
自定义下拉框 - 组件层

设计思路：
- 基于 PopupMenuButton 实现，交互可靠
- 按钮和菜单项高度严格一致，视觉效果统一
- 用户指定变量集中管理，方便调试和维护

功能：
- 支持自定义宽度和高度
- 菜单宽度自适应内容
- 选中后自动关闭
"""
```

## 用户指定变量规则详解

### 优先级顺序
```
调用模块用户指定值 > 被调用模块用户指定默认值 > 配置文件值
```

### 示例

**场景1：使用被调用模块的默认值**
```python
# 自定义下拉框.py
DEFAULT_WIDTH = 150  # 用户指定默认值

# 下拉卡片.py（调用方）
# 未指定width，使用被调用模块的默认值150
dropdown = CustomDropDown(config=config)
```

**场景2：调用方覆盖默认值**
```python
# 自定义下拉框.py
DEFAULT_WIDTH = 150  # 用户指定默认值

# 下拉卡片.py（调用方）
CARD_DROPDOWN_WIDTH = 100  # 用户指定值，覆盖被调用模块的默认值
dropdown = CustomDropDown(config=config, width=CARD_DROPDOWN_WIDTH)  # 最终宽度100
```

### 禁止行为
- ❌ 在同一模块内对用户指定变量进行二次覆盖
- ❌ 使用硬编码值覆盖用户指定变量

## 函数设计
- 函数应只做一件事
- 长度建议不超过20行
- 避免多层嵌套，提前返回

## 类设计
- 相关代码放在一起
- 保持适当抽象层次
- 类超过300行考虑拆分

## 代码坏味道识别
### 重复代码
- 提取为函数或类

### 过长参数列表（>3）
- 引入参数对象

### 数据泥团
- 经常一起出现的字段封装为对象

## 模块划分
- 按功能垂直切分
- 避免循环依赖
- 接口清晰

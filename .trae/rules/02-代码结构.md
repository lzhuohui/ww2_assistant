# 代码结构规范

## 模块结构

每个模块必须包含：
1. **文件头**：编码声明和文档字符串
2. **导入**：必要的模块导入
3. **用户指定变量**：集中管理，AI不得修改
4. **类/函数定义**：核心功能实现
5. **调试逻辑**：标准化测试入口

## 文档字符串

模块开头必须包含：
- 模块名称和层级
- 设计思路（原理、决策点）
- 功能列表

## 用户指定变量

```python
# *** 用户指定变量 - AI不得修改 ***
DEFAULT_WIDTH = 120  # 默认宽度
# *********************************
```

**规则**：
- 不得在同一模块内二次覆盖
- 调用方可以覆盖被调用方默认值
- 优先级：调用方 > 被调用方 > 配置文件

## 调试逻辑

```python
# ==================== 调试逻辑 ====================
if __name__ == "__main__":
    配置 = 界面配置()
    def main(page: ft.Page):
        page.padding = 0
        page.bgcolor = 配置.当前主题颜色["bg_primary"]
        page.add(被测模块(配置).render())
    ft.run(main)
```

**原则**：
- 只能更改被测模块名称
- 禁止硬编码和额外包装
- 统一结构，如实测试

## 代码设计

### 函数
- 只做一件事
- 长度≤20行
- 避免多层嵌套

### 类
- 相关代码集中
- 适当抽象层次
- 超过300行考虑拆分

### 模块划分
- 按功能垂直切分
- 避免循环依赖
- 接口清晰

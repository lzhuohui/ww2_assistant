# 技能体系学习路径

## 学习路径概览

本技能体系采用渐进式学习路径，从基础规则理解到高级问题解决，分为三个阶段：入门、进阶、精通。

```
入门阶段 → 进阶阶段 → 精通阶段
(基础层)   (应用层)   (高级层)
```

---

## 第一阶段：入门阶段（基础层）

### 学习目标
- 理解项目规则体系的整体架构
- 掌握规则查询和导航方法
- 了解规则冲突的基本概念

### 学习时长
建议学习时间：1-2周

### 学习内容

#### 1. 规则体系认知
**学习要点**：
- 三层架构：核心规范层、详细规范层、检查清单层
- 规则配置：alwaysApply、description、globs
- 规则引用机制：#Rule:路径格式

**实践任务**：
- 浏览规则索引文件
- 理解核心规范层的作用
- 尝试查询特定规则

#### 2. 规则查询助手技能
**学习要点**：
- 技能触发条件和使用场景
- 查询结果的解读方法
- 规则引用的使用

**实践任务**：
- 使用规则查询助手查询模块结构规范
- 查询用户变量保护规则
- 查询模块隔离原则

#### 3. 规则索引导航技能
**学习要点**：
- 规则体系的组织结构
- 规则分类和层级关系
- 快速定位规则的方法

**实践任务**：
- 使用规则索引导航浏览规则体系
- 查找特定类型的规则
- 理解规则的优先级

#### 4. 规则冲突检测技能（基础）
**学习要点**：
- 规则冲突的基本类型
- 冲突检测的基本方法
- 冲突解决的基本策略

**实践任务**：
- 检测简单的规则冲突
- 理解冲突报告格式
- 学习基本的冲突解决方法

### 阶段验收标准
- [ ] 能够独立查询和理解规则内容
- [ ] 能够使用规则索引导航定位规则
- [ ] 能够识别简单的规则冲突
- [ ] 理解三层架构的作用和关系

## 第二阶段：进阶阶段（应用层）

### 学习目标
- 掌握规则在实际开发中的应用
- 能够生成符合规范的代码结构
- 能够进行代码审查和合规检查

### 学习时长
建议学习时间：2-3周

### 学习内容

#### 1. 模块结构生成器技能
**学习要点**：
- 模块结构的标准模板
- 用户变量区域的创建
- 调试逻辑的标准格式

**实践任务**：
- 使用模块结构生成器创建单元模块
- 创建组合模块
- 创建页面模块

#### 2. 代码审查助手技能
**学习要点**：
- 代码审查的标准流程
- 模块复用检查方法
- 违规问题的识别和分类

**实践任务**：
- 审查新建模块的代码
- 审查代码修改
- 生成审查报告

#### 3. 模块复用检测器技能
**学习要点**：
- 模块复用优先原则
- 重复代码的识别方法
- 复用机会的发现

**实践任务**：
- 检测项目中的重复代码
- 发现模块复用机会
- 提供复用优化建议

#### 4. 规范合规检查器技能
**学习要点**：
- 合规检查的全面流程
- 违规等级的定义和分类
- 整改建议的制定

**实践任务**：
- 执行代码合规检查
- 生成合规报告
- 制定整改方案

### 阶段验收标准
- [ ] 能够独立生成符合规范的模块结构
- [ ] 能够执行完整的代码审查流程
- [ ] 能够发现模块复用机会并提供建议
- [ ] 能够执行全面的合规检查并生成报告

---

## 第三阶段：精通阶段（高级层）

### 学习目标
- 掌握复杂问题的诊断和分析方法
- 能够设计符合规范的解决方案
- 能够规划和执行代码重构

### 学习时长
建议学习时间：3-4周

### 学习内容

#### 1. 问题诊断专家技能
**学习要点**：
- 问题根因分析方法
- 多维度问题诊断
- 复杂问题的分解策略

**实践任务**：
- 诊断运行时错误
- 分析规范违规问题
- 诊断性能问题

#### 2. 解决方案设计师技能
**学习要点**：
- 方案设计的方法论
- 规范合规验证
- 实施计划的制定

**实践任务**：
- 设计新功能方案
- 优化现有系统
- 重构遗留代码

#### 3. 重构规划师技能
**学习要点**：
- 重构必要性的评估
- 重构风险的控制
- 渐进式重构策略

**实践任务**：
- 规划模块结构重构
- 规划架构优化重构
- 规划代码质量提升

### 阶段验收标准
- [ ] 能够独立诊断复杂问题并找到根本原因
- [ ] 能够设计符合规范的完整解决方案
- [ ] 能够规划安全的重构方案并控制风险
- [ ] 能够指导他人使用技能体系

## 学习建议

### 入门阶段建议
1. **循序渐进**：先理解规则体系，再学习具体技能
2. **多实践**：每个技能都要动手实践，不要只看不练
3. **记录问题**：遇到问题及时记录，便于后续深入学习

### 进阶阶段建议
1. **结合实际项目**：在实际项目中应用技能，加深理解
2. **对比学习**：对比不同技能的使用场景，理解差异
3. **总结经验**：总结使用经验，形成自己的方法论

### 精通阶段建议
1. **深入分析**：不仅知其然，还要知其所以然
2. **举一反三**：将技能应用到更复杂的场景
3. **分享传承**：将经验分享给他人，教学相长

## 能力评估

### 评估维度
1. **知识掌握**：对规则和技能的理解程度
2. **实践能力**：实际应用技能的能力
3. **问题解决**：解决复杂问题的能力
4. **创新能力**：优化和改进的能力

### 评估方法
- 完成验证任务集
- 实际项目应用
- 同行评审
- 自我评估

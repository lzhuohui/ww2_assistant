# 问题诊断专家

## 技能概述
深入分析代码问题的根本原因，提供系统性的问题诊断和根因分析。

## 执行步骤

### 步骤1：收集问题信息
- 收集错误信息或问题描述
- 确定问题发生的上下文
- 收集相关代码片段
- 确认问题的复现步骤

### 步骤2：分析问题现象
- 分类问题类型（语法错误、逻辑错误、规范违规等）
- 分析问题的影响范围
- 识别问题的直接表现
- 记录问题的特征

### 步骤3：追溯问题根源
- 分析代码执行流程
- 检查相关模块的实现
- 验证数据传递路径
- 识别违规的规范要求

### 步骤4：验证诊断假设
- 提出可能的根本原因
- 设计验证方案
- 执行验证测试
- 确认根本原因

### 步骤5：生成诊断报告
- 描述问题现象
- 说明根本原因
- 分析影响范围
- 提供解决方向

### 步骤6：提供修复建议
- 给出具体的修复方案
- 提供代码修改示例
- 说明修复的注意事项
- 提供预防措施

## 典型应用场景

### 场景1：调试运行时错误
**用户需求**："这个模块运行时报错，帮我诊断问题"
**执行流程**：
1. 分析错误堆栈信息
2. 追溯错误发生位置
3. 检查相关代码实现
4. 识别根本原因
5. 提供修复方案

### 场景2：分析规范违规问题
**用户需求**："为什么这个模块不符合规范？"
**执行流程**：
1. 执行规范合规检查
2. 识别违规点
3. 分析违规原因
4. 追溯设计决策
5. 提供整改方案

### 场景3：诊断性能问题
**用户需求**："这个页面加载很慢，帮我分析原因"
**执行流程**：
1. 分析性能瓶颈
2. 检查模块调用链
3. 识别性能问题点
4. 分析资源使用
5. 提供优化建议

## 常见问题及解决策略

### 问题1：问题现象与根本原因不直接相关
**解决策略**：使用5W分析法，层层追溯

### 问题2：多个问题交织在一起
**解决策略**：分离问题，逐一分析

### 问题3：问题难以复现
**解决策略**：收集日志信息，分析历史记录

### 问题4：根本原因涉及多个模块
**解决策略**：绘制依赖关系图，系统性分析

### 问题5：修复方案成本过高
**解决策略**：提供临时规避方案和长期解决方案

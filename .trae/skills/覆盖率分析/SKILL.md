# 测试覆盖率分析器

## 描述
分析项目测试覆盖率，识别未测试的代码路径，提供改进建议。

## 使用场景
- 用户要求"检查测试覆盖率"
- 代码提交前验证
- 6A流程中的Assess阶段调用

## 指令

### 步骤1：运行覆盖率测试
执行 `pytest --cov=模块名 --cov-report=html` 生成覆盖率报告。

### 步骤2：分析结果
- 统计总体覆盖率
- 识别覆盖率<85%的模块
- 标记未测试的函数和分支

### 步骤3：生成改进建议
为低覆盖率模块提供：
- 缺失的测试场景清单
- 边界值测试建议
- 异常处理测试建议

### 步骤4：输出报告
生成 `reports/coverage_report.md`，包含：
- 覆盖率统计表
- 未测试代码清单
- 优先级改进建议

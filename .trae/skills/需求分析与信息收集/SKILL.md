## 阶段1：需求分析与确认

### 1. 需求拆解

**拆解方法**：
- **功能维度**：识别核心功能和辅助功能
- **技术维度**：分析技术需求和依赖
- **业务维度**：理解业务目标和价值
- **约束维度**：识别时间、资源和技术限制

**拆解示例**：
```markdown
## 需求拆解报告

### 核心需求
1. **功能需求**：
   - 数据处理功能
   - 用户界面设计
   - 性能优化

2. **技术需求**：
   - 编程语言：Python
   - 框架：Flet
   - 数据库：SQLite

3. **业务需求**：
   - 提高用户体验
   - 降低系统响应时间
   - 增强数据安全性

4. **约束条件**：
   - 开发时间：2周
   - 预算：有限
   - 技术栈：现有技术栈
```

### 2. 需求澄清

**澄清机制**：
- **识别模糊点**：标记不明确的需求内容
- **提出具体问题**：针对模糊点提出澄清问题
- **获取详细信息**：要求用户提供更多细节
- **确认理解**：确保对需求的理解与用户一致

**澄清示例**：
```markdown
## 需求澄清

### 模糊点识别
1. **功能描述**："数据处理功能"过于概括
2. **性能要求**：未明确具体的性能指标
3. **安全要求**：未说明安全级别和具体措施

### 澄清问题
1. 数据处理功能具体包括哪些操作？（如数据导入、转换、分析等）
2. 系统的响应时间目标是多少？
3. 数据安全需要达到什么级别的保护？
4. 是否有特定的用户界面风格要求？
```

### 3. 需求确认

**确认流程**：
1. 展示拆解结果和澄清后的需求
2. 要求用户确认需求的完整性和准确性
3. 记录用户的确认或修改意见
4. 形成最终的需求文档

**确认示例**：
```markdown
## 需求确认

### 确认内容
1. 功能需求：数据导入、转换、分析和可视化
2. 技术需求：Python + Flet + SQLite
3. 性能要求：响应时间 < 1秒
4. 安全要求：数据加密存储
5. 开发时间：2周

### 用户确认
□ 需求完整且准确
□ 需要修改以下内容：
  _________________________________

### 确认时间
[用户确认时间]
```

## 阶段2：信息收集与最佳实践整合

### 1. 网络信息检索

**检索策略**：
- **关键词提取**：从确认的需求中提取关键概念
- **检索渠道**：技术文档、教程、论坛、案例研究
- **检索范围**：相关技术、设计模式、实现方案
- **结果整理**：分类和组织检索结果

**检索示例**：
```markdown
## 信息检索报告

### 检索关键词
- Python Flet 数据处理
- SQLite 性能优化
- 数据可视化最佳实践
- Python 安全加密

### 检索结果
1. **技术文档**：
   - Flet 官方文档：https://flet.dev/docs
   - SQLite 性能优化指南：https://www.sqlite.org/optoverview.html

2. **教程资源**：
   - Python 数据处理教程：https://realpython.com/
   - Flet 界面设计教程：https://flet.dev/docs/tutorials

3. **案例研究**：
   - 类似项目的实现案例
   - 性能优化成功案例
```

### 2. 最佳实践整合

**整合方法**：
- **模式识别**：识别相关领域的设计模式
- **案例分析**：分析成功实施的案例
- **最佳实践提取**：提取可应用的最佳实践
- **解决方案构建**：基于最佳实践构建解决方案

**最佳实践示例**：
```markdown
## 最佳实践整合

### 设计模式
1. **MVC模式**：分离数据、视图和控制器
2. **工厂模式**：创建对象的最佳实践
3. **单例模式**：管理共享资源

### 实施案例
1. **数据处理**：使用Pandas进行高效数据处理
2. **界面设计**：Flet组件的最佳组合方式
3. **性能优化**：SQLite索引优化策略

### 推荐实践
1. **代码组织**：模块化设计，分离关注点
2. **性能优化**：使用缓存减少数据库查询
3. **安全措施**：使用加密库保护敏感数据
```

### 3. 信息筛选与评估

**筛选标准**：
- **时效性**：优先选择近期的信息
- **权威性**：优先选择官方文档和权威来源
- **适用性**：评估信息与当前需求的相关性
- **可靠性**：验证信息的准确性和可靠性

**评估示例**：
```markdown
## 信息评估报告

### 信息来源评估
1. **官方文档**：✅ 高权威，高适用
2. **技术博客**：⚠️ 中等权威，需验证
3. **论坛讨论**：⚠️ 低权威，参考价值有限

### 信息时效性
1. **近期信息**（1年内）：70%
2. **中期信息**（1-3年）：25%
3. **远期信息**（3年以上）：5%

### 适用性评估
1. **高度相关**：60%
2. **中度相关**：30%
3. **低度相关**：10%
```

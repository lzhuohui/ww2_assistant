# 重构规划师

## 技能概述
规划系统性的代码重构方案，确保重构过程安全可控，重构结果符合规范要求。

## 执行步骤

### 步骤1：评估重构必要性
- 分析代码质量问题
- 评估技术债务
- 确认重构收益
- 识别重构风险

### 步骤2：确定重构范围
- 确定重构的模块或文件
- 分析模块依赖关系
- 评估影响范围
- 制定隔离策略

### 步骤3：设计重构目标
- 明确重构目标
- 定义成功标准
- 设定质量指标
- 规划验收方案

### 步骤4：规划重构步骤
- 拆分重构任务
- 排序任务优先级
- 设计任务依赖
- 估算工作量

### 步骤5：制定安全措施
- 设计测试方案
- 规划回滚策略
- 准备监控方案
- 制定应急预案

### 步骤6：生成重构计划
- 编写重构方案
- 提供代码示例
- 制定时间计划
- 明确验收标准

## 典型应用场景

### 场景1：模块结构重构
**用户需求**："重构这个模块使其符合规范"
**执行流程**：
1. 分析现有模块结构
2. 识别违规问题
3. 设计目标结构
4. 规划重构步骤
5. 提供代码示例

### 场景2：架构优化重构
**用户需求**："优化这个子系统的架构"
**执行流程**：
1. 分析现有架构
2. 识别架构问题
3. 设计目标架构
4. 规划迁移路径
5. 提供实施方案

### 场景3：代码质量提升
**用户需求**："提升这个模块的代码质量"
**执行流程**：
1. 执行质量检查
2. 识别质量问题
3. 设计改进方案
4. 规划改进步骤
5. 提供代码示例

## 常见问题及解决策略

### 问题1：重构风险过高
**解决策略**：采用渐进式重构，分阶段实施

### 问题2：重构影响面过大
**解决策略**：使用适配器模式隔离影响

### 问题3：测试覆盖不足
**解决策略**：先补充测试，再执行重构

### 问题4：重构时间窗口有限
**解决策略**：优先处理高风险问题，分批实施

### 问题5：团队技能不足
**解决策略**：提供详细的重构指导和代码示例

# 规则变更报告生成器

## 技能概述

在进行规则更改操作时，自动生成详细的对比评估报告，确保用户在明确确认后才能执行修改。

## 触发条件

1. AI自主发现规则错误
2. 用户主动要求规则修改

## 报告模板

### 规则变更评估报告

```markdown
## 规则变更评估报告

### 变更概要
- **变更类型**: [新增/修改/删除]
- **触发来源**: [AI自主发现/用户主动要求]
- **规范文件**: [文件路径]
- **变更时间**: [YYYY-MM-DD HH:mm:ss]

### 变更内容对比

#### 变更前
```
[原规范内容]
```

#### 变更后
```
[新规范内容]
```

#### 变更差异
| 行号 | 变更类型 | 原内容 | 新内容 |
|------|----------|--------|--------|
| XX | 修改/新增/删除 | ... | ... |

### 影响分析

#### 1. 对其他规范的影响
- [ ] 无影响
- [ ] 有影响，具体如下：
  - 规范A：[影响说明]
  - 规范B：[影响说明]

#### 2. 对现有代码的影响
- [ ] 无影响
- [ ] 有影响，涉及文件数：X个
  - 文件1：[影响说明]
  - 文件2：[影响说明]

#### 3. 对AI执行的影响
- [ ] 无影响
- [ ] 有影响，具体如下：
  - [影响说明]

### 风险评估

| 风险项 | 风险等级 | 说明 |
|--------|----------|------|
| 规范一致性 | 低/中/高 | [说明] |
| 代码合规性 | 低/中/高 | [说明] |
| AI理解准确性 | 低/中/高 | [说明] |
| 用户体验 | 低/中/高 | [说明] |

**综合风险等级**: 低/中/高

### 最优修改建议

#### 建议方案
[基于准确性优化目标的最优修改建议]

#### 替代方案（如有）
[其他可行的修改方案]

#### 不修改的影响
[如果不进行此修改，可能产生的影响]

### 确认选项

请选择：
1. **确认执行** - 按建议方案修改
2. **采用替代方案** - 按替代方案修改
3. **暂不修改** - 取消本次变更
4. **查看详情** - 查看更多分析信息

### 变更记录
- 本次变更将由系统自动记录
- 变更原因：[记录变更原因]
- 确认人：[用户标识]
```

## 分级处理机制

### 低风险变更（直接执行后记录）

| 条件 | 处理方式 |
|------|----------|
| 新增规范（不与现有规范冲突） | 简化报告 + 自动执行 + 记录 |
| 完善规范（补充说明，不改变含义） | 简化报告 + 自动执行 + 记录 |
| 修正错别字/格式问题 | 自动执行 + 记录 |

### 中风险变更（简化确认）

| 条件 | 处理方式 |
|------|----------|
| 修改规范（改变约束条件） | 完整报告 + 用户确认 |
| 删除规范（非核心规范） | 完整报告 + 用户确认 |

### 高风险变更（完整确认）

| 条件 | 处理方式 |
|------|----------|
| 修改核心规范 | 完整报告 + 强调风险 + 用户确认 |
| 删除核心规范 | 完整报告 + 强调风险 + 用户确认 |
| 规范间产生矛盾 | 完整报告 + 冲突说明 + 用户确认 |

## 变更记录存储

### 记录格式

```json
{
  "change_id": "RULE_20240101_001",
  "timestamp": "2024-01-01T12:00:00Z",
  "trigger_source": "user_request",
  "file_path": ".trae/rules/xxx.md",
  "change_type": "modify",
  "risk_level": "medium",
  "summary": "变更摘要",
  "before": "原内容",
  "after": "新内容",
  "impact_analysis": {
    "other_rules": [],
    "affected_files": [],
    "ai_execution": "无影响"
  },
  "confirmed_by": "user",
  "confirmed_at": "2024-01-01T12:05:00Z"
}
```

### 存储位置

- 变更记录文件：`.trae/logs/rule_changes.jsonl`
- 每次变更追加一条记录

## AI执行检查清单

在生成规则变更报告时，AI必须检查：

- [ ] 是否识别了变更类型？
- [ ] 是否进行了完整的影响分析？
- [ ] 是否评估了风险等级？
- [ ] 是否提供了最优修改建议？
- [ ] 是否等待用户确认后才执行？

## 技能元数据

- **技能类型**：审核技能
- **触发方式**：自动触发
- **适用对象**：规范管理者
- **前置条件**：规范变更审核

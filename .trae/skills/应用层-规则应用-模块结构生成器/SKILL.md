# 模块结构生成器

## 技能概述
根据项目规范自动生成符合标准的模块结构，确保新模块从一开始就符合所有规范要求。

## 执行步骤

### 步骤1：收集模块需求
- 确认模块名称和功能
- 确定模块类型（单元模块、组合模块、页面模块）
- 收集模块参数需求
- 确认是否需要用户指定变量区域

### 步骤2：加载结构规范
- 读取核心规范层/代码结构规范_精简版.md
- 读取详细规范层/代码结构规范/01_模块结构规范.md
- 读取详细规范层/代码结构规范/02_文档字符串规范.md
- 读取详细规范层/代码结构规范/03_用户指定变量规范.md

### 步骤3：生成模块结构
- 生成文件头（编码声明、文档字符串）
- 创建导入区域（标准库、第三方库、项目模块）
- 创建用户指定变量区域
- 生成类/函数定义框架
- 添加调试逻辑模板

### 步骤4：填充默认内容
- 添加默认参数值
- 生成文档字符串模板
- 添加类型注解
- 创建返回语句框架

### 步骤5：验证结构合规
- 检查模块结构完整性
- 验证文档字符串格式
- 确认用户变量区域存在
- 验证导入顺序正确

### 步骤6：输出模块文件
- 生成完整的模块代码
- 提供使用说明
- 标注需要用户填充的部分
- 提供相关规范引用

## 常见问题及解决策略

### 问题1：模块名称不规范
**解决策略**：自动转换为PascalCase格式，提示用户确认

### 问题2：参数类型不明确
**解决策略**：提供常见参数类型建议，使用Any作为默认类型

### 问题3：用户变量区域需求不明确
**解决策略**：默认创建空的用户变量区域，添加注释说明

### 问题4：导入依赖不确定
**解决策略**：根据模块类型推断必要导入，提供可选导入列表

### 问题5：调试逻辑不符合实际需求
**解决策略**：提供标准调试模板，允许用户自定义修改

## 输出规范

### 模块文件格式
```python
# -*- coding: utf-8 -*-
"""模块名称：[ModuleName] | [设计思路] | 模块隔离原则"""
import flet as ft

# *** 用户指定变量 - AI不得修改 ***
# （用户未指定变量）
# *********************************

class [ModuleName]:
    """[模块功能说明]"""
    
    @staticmethod
    def create([参数列表]) -> ft.Container:
        """[方法说明]
        
        Args:
            [参数说明]
        
        Returns:
            ft.Container: [返回值说明]
        """
        return ft.Container()

if __name__ == "__main__":
    ft.run(lambda page: page.add([ModuleName].create()))
```

### 输出文件命名
- 文件名：模块名称转换为snake_case
- 位置：根据模块类型确定目录
- 编码：UTF-8

# 规则冲突检测

## 技能概述
检测项目规则体系中存在的冲突和矛盾，确保规则的一致性和可执行性。

## 执行步骤

### 步骤1：收集相关规则
- 根据用户指定的规则范围收集规则文件
- 提取规则的约束条件
- 识别规则的适用场景
- 记录规则的优先级

### 步骤2：分析规则冲突
- 检测相互矛盾的约束
- 识别重叠的适用场景
- 发现优先级冲突
- 检查引用链完整性

### 步骤3：评估冲突影响
- 分析冲突的严重程度
- 评估对项目的影响范围
- 识别受影响的代码模块
- 评估解决冲突的难度

### 步骤4：生成冲突报告
- 列出所有检测到的冲突
- 提供冲突的详细说明
- 给出冲突解决建议
- 提供优先级排序

### 步骤5：提供解决方案
- 给出规则修改建议
- 提供规则合并方案
- 建议规则拆分策略
- 提供临时规避方案

## 典型应用场景

### 场景1：检测命名规范冲突
**用户需求**："检查命名规范是否存在冲突"
**执行流程**：
1. 收集所有命名相关规则
2. 检测命名格式要求冲突
3. 识别命名范围重叠
4. 提供统一命名建议

### 场景2：验证模块结构规范一致性
**用户需求**："验证模块结构规范是否一致"
**执行流程**：
1. 收集核心规范和详细规范
2. 对比结构要求
3. 检测矛盾点
4. 提供一致性建议

### 场景3：合并多个规则
**用户需求**："合并这两个规则文件"
**执行流程**：
1. 分析两个规则的内容
2. 检测冲突点
3. 提供合并方案
4. 生成新的规则文件

## 常见问题及解决策略

### 问题1：隐式冲突难以发现
**解决策略**：使用语义分析，检测深层冲突

### 问题2：优先级不明确
**解决策略**：建立明确的优先级规则

### 问题3：冲突解决成本高
**解决策略**：提供渐进式解决方案

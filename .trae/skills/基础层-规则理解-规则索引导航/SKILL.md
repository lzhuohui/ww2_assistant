# 规则索引导航

## 技能概述
提供项目规则体系的完整导航，帮助用户快速定位和理解规则的组织结构。

## 执行步骤

### 步骤1：加载规则索引
- 读取文档报告/规则索引.md
- 解析三层架构结构
- 提取各层级规则列表
- 构建规则树状结构

### 步骤2：构建导航视图
- 按层级组织规则目录
- 标注规则配置信息（alwaysApply、description）
- 显示规则文件路径
- 提供规则统计信息

### 步骤3：提供搜索功能
- 支持关键词搜索
- 支持规则类型筛选
- 支持层级筛选
- 支持路径搜索

### 步骤4：生成导航报告
- 输出规则体系总览
- 提供各层级详细列表
- 标注重要规则
- 提供快速访问链接

## 典型应用场景

### 场景1：浏览规则体系结构
**用户需求**："我想了解项目的规则体系结构"
**执行流程**：
1. 读取规则索引文件
2. 展示三层架构
3. 列出各层级包含的规则
4. 提供规则数量统计

### 场景2：查找特定类型的规则
**用户需求**："查找所有代码结构相关的规则"
**执行流程**：
1. 搜索包含"代码结构"关键词的规则
2. 按层级分类展示
3. 提供规则路径和描述
4. 给出快速访问方式

### 场景3：了解核心规范
**用户需求**："哪些是核心规范？"
**执行流程**：
1. 筛选核心规范层规则
2. 标注alwaysApply: true的规则
3. 列出核心规范清单
4. 说明核心规范的重要性

## 常见问题及解决策略

### 问题1：规则索引文件不存在
**解决策略**：扫描规则目录，自动生成索引

### 问题2：规则索引过期
**解决策略**：对比实际文件和索引，更新索引

### 问题3：规则分类不清晰
**解决策略**：提供多维度分类视图

## 输出规范

### 导航报告格式
```
## 规则体系导航

### 核心规范层（始终生效）
1. [规则名称](路径) - 描述
2. ...

### 详细规范层（智能匹配）
#### 子分类1
1. [规则名称](路径) - 描述
2. ...

### 检查清单层（手动触发）
1. [清单名称](路径) - 描述
2. ...

### 统计信息
- 总规则数：XX
- 核心规范：XX
- 详细规范：XX
- 检查清单：XX
```

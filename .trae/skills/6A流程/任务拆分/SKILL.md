# 6A阶段3：Atomize（原子化拆分）

## 描述
将架构设计中的功能模块拆分为可独立开发测试的原子任务。

## 使用场景
- 6A总控调用
- 用户要求"拆分任务"

## 指令

### 步骤1：解析架构文档
读取 `DESIGN_任务名.md`，识别需要开发的组件。

### 步骤2：定义模块契约（关键）
在实现前，必须先定义模块接口契约：
- 创建 `contracts/模块名-interface.md`
- 定义函数签名、前置条件、后置条件
- 定义性能约束
- 提交用户审查并锁定

### 步骤3：生成TASKS文档
在 `docs/任务名/TASKS_任务名.md` 中列出所有子任务，每个任务包含：
- 任务ID、名称
- 输入契约、输出契约
- 技术依赖
- 预估工时（1-8小时）
- 验收标准
- **模块状态跟踪**：开发中 → 测试中 → 已冻结

### 步骤4：确定开发顺序（防止死循环）
按依赖关系排序，确保：
- 底层模块优先开发
- 上层模块只能调用已冻结的下层模块
- 禁止反向修改（解决A时不得修改B）

### 步骤5：生成依赖图
使用Mermaid绘制任务依赖关系，标注：
- 模块间的调用关系
- 开发顺序（序号）
- 契约状态

### 步骤6：输出
展示任务列表、依赖图和契约文档，并提示：
"任务拆分完成，请确认工时、依赖和契约。注意：模块一旦冻结，禁止修改！"

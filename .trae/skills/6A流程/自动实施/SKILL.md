# 6A阶段5：Automate（自动化实施）

## 描述
在编码阶段通过自动化工具提升效率，确保代码质量符合规范。

## 使用场景
- 6A总控调用
- 用户开始编码时

## 指令

### 步骤1：生成代码骨架
根据TASK文档中的输入输出契约，生成符合项目规范的代码：
- Controller层（参数校验）
- Service层（业务逻辑）
- Repository层（数据访问）
- 单元测试模板

### 步骤2：实时代码检查
- 安全：无硬编码密钥（从环境变量读取）
- 异常处理：关键路径添加try-catch、重试
- 命名规范：符合PEP 8，避免"-er"后缀
- 注释：解释"为什么"

### 步骤3：自动生成配置文件
根据项目类型生成 Dockerfile、docker-compose.yml、CI/CD配置。

### 步骤4：文档同步检查
代码提交时，提醒更新对应的设计文档或API文档。

### 步骤5：输出
代码实施完成后，汇总："代码已生成，覆盖率≥85%，已配置CI/CD。"

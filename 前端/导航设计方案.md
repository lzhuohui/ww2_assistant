# Windows 11风格导航设计方案

## 一、Windows 11设置界面设计分析

### 1. 视觉设计语言
- **色彩系统**：深色主题为主，使用#1C1C1C作为主背景，#2D2D2D作为卡片背景，#0078D4作为强调色
- **字体样式**：使用无衬线字体，清晰易读，层次分明
- **控件形态**：圆角设计，柔和的阴影效果，扁平化风格
- **空间关系**：充足的留白，清晰的视觉层次，模块化布局

### 2. 布局结构
- **左侧导航**：垂直排列的导航项，带有分组和缩进
- **右侧内容**：卡片式布局，内容区域清晰分明
- **顶部标题**：显示当前页面标题，简洁明了
- **响应式设计**：在不同屏幕尺寸下自适应调整

### 3. 交互模式
- **导航切换**：点击导航项时平滑过渡到对应内容
- **悬停效果**：导航项有微妙的悬停效果
- **选中状态**：选中的导航项有明显的视觉反馈
- **滚动行为**：内容区域滚动时导航保持固定

## 二、旧版导航功能数据项梳理

### 1. 现有导航项
- **系统设置**：显示设置、声音设置、主题设置
- **设备设置**：ADB设备管理、设备信息
- **游戏设置**：游戏路径设置、游戏相关设置
- **脚本设置**：脚本运行参数、安全设置
- **监控面板**：运行状态、运行日志

### 2. 分类逻辑
- 按功能模块划分，每个模块包含相关的设置选项
- 层次结构清晰，便于用户快速定位所需功能
- 功能关联性强，相关设置集中在一起

## 三、布局方案设计

### 方案一：经典Windows 11导航布局

#### 结构说明
- **左侧导航栏**：固定宽度240px，包含主导航项和子导航项
- **右侧内容区**：自适应宽度，卡片式布局
- **顶部状态栏**：显示应用标题和基本信息

#### 视觉设计规范
- 导航项高度48px，间距8px
- 子导航项缩进24px，使用较小字体
- 选中状态使用强调色背景和文字
- 卡片使用8px圆角，轻微阴影效果

#### 交互流程图
1. 用户打开应用 → 显示默认选中的导航项内容
2. 用户点击导航项 → 切换到对应内容，更新选中状态
3. 用户调整窗口大小 → 布局自适应调整
4. 用户切换主题 → 整个界面颜色方案更新

### 方案二：紧凑式导航布局

#### 结构说明
- **左侧导航栏**：固定宽度200px，仅显示主导航项
- **右侧内容区**：更大的内容区域，卡片式布局
- **顶部导航**：在内容区顶部显示当前模块的子导航

#### 视觉设计规范
- 导航项高度44px，间距6px
- 顶部子导航使用标签页形式
- 卡片使用6px圆角，更紧凑的布局
- 减少留白，增加信息密度

#### 交互流程图
1. 用户打开应用 → 显示默认选中的导航项内容
2. 用户点击主导航项 → 切换到对应模块，顶部显示子导航
3. 用户点击顶部子导航 → 切换到对应子内容
4. 用户调整窗口大小 → 布局自适应调整

### 方案三：响应式导航布局

#### 结构说明
- **左侧导航栏**：在宽屏模式下显示完整导航
- **顶部导航栏**：在窄屏模式下自动切换为顶部导航
- **内容区域**：根据屏幕尺寸自动调整布局

#### 视觉设计规范
- 宽屏模式：左侧导航240px，右侧内容自适应
- 中屏模式：左侧导航200px，右侧内容自适应
- 窄屏模式：顶部导航，内容区域全屏
- 卡片布局根据屏幕宽度自动调整列数

#### 交互流程图
1. 用户打开应用 → 根据屏幕尺寸显示合适的导航布局
2. 用户调整窗口大小 → 导航布局自动切换
3. 用户点击导航项 → 切换到对应内容
4. 用户切换主题 → 整个界面颜色方案更新

## 四、可访问性与扩展性考虑

### 1. 可访问性
- 支持键盘导航和屏幕阅读器
- 合理的颜色对比度，确保文字清晰可读

### 2. 易用性
- 导航结构清晰，层次分明
- 常用功能易于访问

- 提供搜索功能，快速定位设置项

### 3. 扩展性
- 模块化设计，便于添加新的导航项和功能
- 主题系统可扩展，支持自定义主题
- 布局系统可适应不同的内容类型
- 支持插件和扩展功能

## 五、设计规范文档

### 1. 组件库定义

#### 导航项组件
- **主导航项**：高度48px，字体大小14px，字重500
- **子导航项**：高度44px，字体大小13px，字重400，缩进24px
- **导航分组标题**：高度32px，字体大小12px，字重600，大写

#### 卡片组件
- **标准卡片**：圆角8px，背景色#2D2D2D（深色主题），#FFFFFF（浅色主题）
- **标题**：字体大小18px，字重600，颜色#F2F2F2（深色主题），#1F2937（浅色主题）
- **副标题**：字体大小12px，字重400，颜色#CCCCCC（深色主题），#6B7280（浅色主题）
- **内容间距**：内部 padding 20px，卡片间距16px

#### 按钮组件
- **主要按钮**：高度36px，圆角4px，背景色#0078D4，文字颜色白色
- **次要按钮**：高度36px，圆角4px，背景色#2D2D2D（深色主题），#FFFFFF（浅色主题），文字颜色#F2F2F2（深色主题），#1F2937（浅色主题）
- **文本按钮**：无背景，文字颜色#0078D4，悬停时显示下划线

### 2. 布局规则

#### 网格系统
- 基础网格单位：4px
- 内容区域最大宽度：1200px
- 侧边栏宽度：240px（宽屏），200px（中屏）
- 响应式断点：768px（平板），1024px（桌面）

#### 间距规范
- 导航项间距：8px
- 卡片间距：16px
- 组件间距：12px
- 内边距：20px

### 3. 色彩方案

#### 深色主题
- **背景色**：#1C1C1C（主背景），#2D2D2D（卡片背景）
- **文字颜色**：#F2F2F2（主要文字），#CCCCCC（次要文字）
- **强调色**：#0078D4（主要强调），#1E3A5F（次要强调）
- **边框颜色**：#404040
- **功能色**：#10B981（成功），#F59E0B（警告），#EF4444（错误）

#### 浅色主题
- **背景色**：#F3F4F6（主背景），#FFFFFF（卡片背景）
- **文字颜色**：#1F2937（主要文字），#6B7280（次要文字）
- **强调色**：#0078D4（主要强调），#E1EBF7（次要强调）
- **边框颜色**：#E5E7EB
- **功能色**：#10B981（成功），#F59E0B（警告），#EF4444（错误）

### 4. 交互规范

#### 导航交互
- **点击**：切换到对应内容，更新选中状态
- **悬停**：导航项背景色变为#3A3A3A（深色主题），#F3F4F6（浅色主题）
- **选中**：导航项背景色变为#1E3A5F（深色主题），#E1EBF7（浅色主题），文字颜色变为#0078D4

#### 内容交互
- **卡片悬停**：轻微阴影效果增强
- **按钮交互**：点击时有轻微的缩放效果
- **输入控件**：获得焦点时有蓝色边框高亮

#### 动画效果
- **导航切换**：内容区域淡入淡出，过渡时间200ms
- **主题切换**：整个界面平滑过渡，过渡时间300ms
- **滚动效果**：平滑滚动，惯性效果

## 六、实施方案

### 1. 代码结构
- **导航组件**：单独的导航组件文件，包含导航项和导航栏实现
- **布局组件**：主布局文件，处理整体布局结构
- **主题系统**：现有的主题系统，确保导航组件支持主题切换
- **响应式处理**：使用Flet的响应式布局功能，处理不同屏幕尺寸

### 2. 实现步骤
1. **创建导航组件**：实现导航项、导航栏和子导航功能
2. **更新主布局**：集成新的导航组件到主布局中
3. **实现响应式设计**：添加屏幕尺寸检测和布局调整逻辑
4. **测试和优化**：在不同屏幕尺寸下测试布局效果，优化交互体验
5. **文档更新**：更新组件文档和使用说明

### 3. 预期效果
- 符合Windows 11设计风格的导航界面
- 响应式布局，适应不同屏幕尺寸
- 流畅的交互体验和动画效果
- 良好的可访问性和扩展性
- 完整的主题支持，支持深色和浅色模式

## 七、结论

通过以上设计方案，我们可以实现一个既符合Windows 11设计规范又能完整呈现旧版导航功能数据的导航界面。三种布局方案各有特点，可以根据具体的使用场景和用户需求选择最合适的方案。

推荐使用**方案三：响应式导航布局**，因为它能够适应不同的屏幕尺寸，提供最佳的用户体验，同时保持Windows 11的设计风格一致性。

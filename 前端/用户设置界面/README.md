# 用户设置界面

## 目录说明

本目录用于开发用户设置界面功能，严格遵循模块化设计原则。

## 模块结构

```
用户设置界面/
├── 文档/                    # 设计文档、评估报告
├── 主界面/                  # 主入口（用户信息卡片、导航栏、主框架）
├── 设置页面/                # 各功能设置页面
├── 组件/                    # 可复用组件
├── 零件/                    # 基础UI零件
├── 工具/                    # 工具函数
└── 配置/                    # 配置文件
```

│                          ↓                                   │
│                         组件                                 │
│   通用卡片.py、卡片容器.py、...                              │
│                          ↓                                   │

## 开发规范

### 1. 依赖原则

- ✅ **高层依赖低层**：主界面→设置页面→组件→零件→配置
- ❌ **禁止反向依赖**：低层不能依赖高层
- ❌ **禁止循环依赖**：模块间不能相互依赖

### 2. 单一职责原则

- 每个模块只负责一个功能
- 模块内部高内聚，模块之间低耦合

### 3. 开闭原则

- 对扩展开放：通过配置扩展新功能
- 对修改关闭：新增功能不修改现有代码

### 4. 接口隔离原则

- 接口最小化设计
- 不强迫模块依赖不需要的接口

## 各层职责

| 层级 | 职责 | 示例模块 |
|------|------|---------|
| 主界面 | 主入口、用户信息、导航 | 主界面.py、用户信息卡片.py、导航栏.py |
| 设置页面 | 各功能设置页面 | 系统设置.py、策略设置.py、账号设置.py |
| 组件 | 可复用组件 | 通用卡片.py、卡片容器.py |
| 零件 | 基础UI元素 | 头像.py、导航按钮.py、标签输入框.py |
| 工具 | 通用工具函数 | 输入验证.py、数据绑定.py |
| 配置 | 配置数据、主题定义 | 界面配置.py、主题配置.py |

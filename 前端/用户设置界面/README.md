# 用户设置界面

## 目录说明

本目录用于开发用户设置界面功能，严格遵循模块化设计原则。

## 模块结构

```
用户设置界面/
├── 文档/            # 设计文档、评估报告
├── 页面层/          # 各功能页面
├── 组件层/          # 可复用组件
├── 零件层/          # 基础UI零件
├── 工具层/          # 工具函数
└── 配置/            # 配置文件
```

## 层级依赖关系

```
┌─────────────────────────────────────────────────────────────┐
│                         页面层                               │
│   主界面.py、各设置页面.py                                   │
│                          ↓                                   │
│                         组件层                               │
│   通用卡片.py、导航栏.py、用户信息卡片.py                    │
│                          ↓                                   │
│                         零件层                               │
│   控件工厂.py、图标标题.py、标签下拉框.py、...               │
│                          ↓                                   │
│                         配置层                               │
│   界面配置.py、配置管理器.py、各配置文件.py                  │
└─────────────────────────────────────────────────────────────┘

工具层：被各层调用（横向依赖）
```

## 开发规范

### 1. 依赖原则

- ✅ **高层依赖低层**：页面层→组件层→零件层→配置层
- ❌ **禁止反向依赖**：低层不能依赖高层
- ❌ **禁止循环依赖**：模块间不能相互依赖

### 2. 单一职责原则

- 每个模块只负责一个功能
- 模块内部高内聚，模块之间低耦合

### 3. 开闭原则

- 对扩展开放：通过配置扩展新功能
- 对修改关闭：新增功能不修改现有代码

### 4. 接口隔离原则

- 接口最小化设计
- 不强迫模块依赖不需要的接口

## 各层职责

| 层级 | 职责 | 示例模块 |
|------|------|---------|
| 页面层 | 页面布局、页面间导航 | 主界面.py、系统设置页面.py |
| 组件层 | 组装零件、协调交互 | 通用卡片.py、导航栏.py |
| 零件层 | 基础UI元素、单一功能 | 标签下拉框.py、卡片容器.py |
| 工具层 | 通用工具函数 | 输入验证.py、数据绑定.py |
| 配置层 | 配置数据、主题定义 | 界面配置.py、主题配置.py |

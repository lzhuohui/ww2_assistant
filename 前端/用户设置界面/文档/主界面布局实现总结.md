# 用户设置界面布局实现总结

## 一、Stack 绝对定位自适应布局（核心经验）

### 1. 问题背景

当使用 Flet 的 `Stack` 进行绝对定位布局时，需要让通用容器自适应主窗口尺寸，同时保持内部组件的对齐关系。

### 2. 关键技术点

#### 2.1 定位属性优先级

在 Flet Stack 中，组件定位有以下规则：

| 场景 | 定位方式 | 说明 |
|------|----------|------|
| **固定尺寸组件** | `left` + `top` + `width` + `height` | 有明确尺寸，只需指定位置 |
| **单边自适应** | `left` + `top` + `width` + `bottom` | 高度自适应，宽度固定 |
| **双边自适应** | `left` + `top` + `right` + `bottom` + `width=None` + `height=None` | 必须清除默认宽高 |

#### 2.2 ⚠️ 重要陷阱：`width` 和 `right` 的冲突

```python
# ❌ 错误：同时设置 width 和 right 会导致 right 失效
nav_interface.width = 280
nav_interface.right = 10  # 无效！width 会覆盖 right 的效果

# ✅ 正确方案A：使用固定宽度，不用 right
nav_interface.width = 280
nav_interface.left = 10  # 只用 left 定位

# ✅ 正确方案B：使用 left/right 双锚定，必须清除 width
nav_interface.width = None
nav_interface.left = 10
nav_interface.right = 10
```

#### 2.3 ⚠️ 重要陷阱：自适应容器中的 `right` 计算问题

当容器自适应主窗口时，Stack 的总宽度是动态的，直接计算 `right` 值会出错：

```python
# ❌ 错误：假设 Stack 宽度为固定值
nav_right = user_width + DEFAULT_SPACING  # right 值计算错误

# ✅ 正确：对齐组件使用相同宽度，避免使用 right
nav_width = user_width  # 与用户界面同宽
nav_interface = NavBar.create(width=nav_width)
nav_interface.left = user_left  # 只用 left 定位
```

### 3. 正确的自适应布局方案

#### 3.1 布局结构

```
主窗口 (Page)
└── 外层容器 (padding=DEFAULT_SPACING)
    └── 通用容器 (expand=True, 自适应)
        └── Stack (expand=True)
            ├── 用户界面 (固定尺寸, left/top 定位)
            ├── 导航界面 (固定宽度, left/top/bottom 定位)
            └── 功能界面 (自适应尺寸, left/top/right/bottom 定位)
```

#### 3.2 组件定位策略

| 组件 | 定位属性 | 宽度策略 | 高度策略 |
|------|----------|----------|----------|
| 用户界面 | `left`, `top` | 固定宽度 | 固定高度 |
| 导航界面 | `left`, `top`, `bottom` | 固定宽度（与用户界面同宽） | 自适应高度 |
| 功能界面 | `left`, `top`, `right`, `bottom` | 自适应宽度 | 自适应高度 |

#### 3.3 关键代码模式

```python
# 用户界面：固定尺寸
user_interface = UserInfoCard.create(width=user_width, height=user_height)
user_interface.left = DEFAULT_SPACING
user_interface.top = DEFAULT_SPACING

# 导航界面：宽度对齐，高度自适应
nav_interface = NavBar.create(width=user_width)  # 与用户界面同宽
nav_interface.left = DEFAULT_SPACING
nav_interface.top = user_height + 2 * DEFAULT_SPACING
nav_interface.bottom = DEFAULT_SPACING

# 功能界面：完全自适应
content_interface = ContentArea.create()
content_interface.width = None   # 必须清除默认宽度
content_interface.height = None  # 必须清除默认高度
content_interface.left = user_width + 2 * DEFAULT_SPACING
content_interface.top = DEFAULT_SPACING
content_interface.right = DEFAULT_SPACING
content_interface.bottom = DEFAULT_SPACING
```

### 4. 8点布局规则实现

```
┌─────────────────────────────────────────────────────────────┐
│ 通用容器 (expand=True)                                        │
│  ┌─────────────┐                                            │
│  │ 用户界面     │ 间距  功能界面                              │
│  │ (固定尺寸)   │ ←→   (left/top/right/bottom)              │
│  │             │                                            │
│  └─────────────┘                                            │
│  ┌─────────────┐                                            │
│  │ 导航界面     │                                            │
│  │ (宽度对齐)   │                                            │
│  │ left/bottom │                                            │
│  └─────────────┘                                            │
│  ←─ 间距 ─→                                                  │
└─────────────────────────────────────────────────────────────┘
```

**8点规则**：
1. 用户界面的左/上边缘 → 距通用容器左/上边缘 1个间距
2. 导航界面上边缘 → 距用户界面下边缘 1个间距
3. 导航界面左/右边缘 → 与用户界面左/右边缘对齐（使用相同宽度实现）
4. 导航界面下边缘 → 距通用容器下边缘 1个间距
5. 功能界面左边缘 → 距用户界面右边缘 1个间距
6. 功能界面上边缘 → 与用户界面上边缘对齐
7. 功能界面下边缘 → 与导航界面下边缘对齐
8. 功能界面右边缘 → 距通用容器右边缘 1个间距

## 三、注意事项清单

### ✅ 必须遵守

1. **清除默认尺寸**：使用 `left/right/bottom` 定位时，必须将对应的 `width/height` 设为 `None`
2. **宽度对齐策略**：需要左右对齐的组件，使用相同宽度 + 只用 `left` 定位，避免使用 `right`
3. **边框一致性**：所有容器的 `border_width` 必须一致（当前为 1）
4. **间距统一**：所有间距使用同一个变量 `DEFAULT_SPACING`

### ❌ 避免的错误

1. 同时设置 `width` 和 `right` 属性
2. 在自适应容器中硬编码计算 `right` 值
3. 不同组件使用不同的边框宽度
4. 组件内部锁定默认尺寸导致外部定位失效

# 界面模块化重构实施细节

## 一、重构目标

1. **Provider模式**：统一管理主题、配置、状态
2. **零件层纯化**：移除组合件，只保留基础UI控件
3. **组件层组合**：负责组合零件，处理简单交互
4. **主界面拆分**：拆分为三大模块

## 三、核心Provider设计

### 3.1 ThemeProvider（主题提供者）

```python
# 核心/主题提供者.py

class ThemeProvider:
    """主题提供者 - 统一管理主题"""
    
    _instance = None
    _theme_colors = None
    _sizes = None
    _listeners = []
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance
    
    @classmethod
    def initialize(cls, config):
        """初始化主题数据"""
        cls._theme_colors = config.当前主题颜色
        cls._sizes = config.定义尺寸
    
    @classmethod
    def get_color(cls, color_name: str) -> str:
        """获取颜色"""
        return cls._theme_colors.get(color_name, "#000000")
    
    @classmethod
    def get_size(cls, category: str, key: str):
        """获取尺寸"""
        return cls._sizes.get(category, {}).get(key)
    
    @classmethod
    def set_theme(cls, theme_colors: dict):
        """设置主题（自动通知所有监听者）"""
        cls._theme_colors = theme_colors
        cls._notify_all()
    
    @classmethod
    def subscribe(cls, listener):
        """订阅主题变化"""
        cls._listeners.append(listener)
    
    @classmethod
    def _notify_all(cls):
        """通知所有监听者"""
        for listener in cls._listeners:
            if hasattr(listener, 'on_theme_change'):
                listener.on_theme_change(cls._theme_colors)
```

### 3.2 ConfigProvider（配置提供者）

```python
# 核心/配置提供者.py

class ConfigProvider:
    """配置提供者 - 统一管理配置"""
    
    _instance = None
    _config_manager = None
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance
    
    @classmethod
    def initialize(cls, config_manager):
        """初始化配置管理器"""
        cls._config_manager = config_manager
    
    @classmethod
    def get_value(cls, card_name: str, config_key: str, default=None):
        """获取配置值"""
        return cls._config_manager.get_value(card_name, config_key, default)
    
    @classmethod
    def set_value(cls, card_name: str, config_key: str, value):
        """设置配置值（即时保存）"""
        cls._config_manager.set_value(card_name, config_key, value)
```

### 3.3 StateProvider（状态提供者）

```python
# 核心/状态提供者.py

class StateProvider:
    """状态提供者 - 统一管理全局状态"""
    
    _instance = None
    _states = {}
    _listeners = {}
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance
    
    @classmethod
    def get_state(cls, key: str, default=None):
        """获取状态"""
        return cls._states.get(key, default)
    
    @classmethod
    def set_state(cls, key: str, value):
        """设置状态（自动通知监听者）"""
        cls._states[key] = value
        if key in cls._listeners:
            for listener in cls._listeners[key]:
                if hasattr(listener, 'on_state_change'):
                    listener.on_state_change(key, value)
    
    @classmethod
    def subscribe(cls, key: str, listener):
        """订阅状态变化"""
        if key not in cls._listeners:
            cls._listeners[key] = []
        cls._listeners[key].append(listener)
```

## 四、零件层基础控件设计

### 4.1 零件层设计原则

```
零件层设计原则：
┌─────────────────────────────────────────────────────────────┐
│ 1. 纯UI：不包含业务逻辑                                     │
│ 2. 无状态：状态由外部管理                                   │
│ 3. 可配置：通过参数配置外观和行为                           │
│ 4. 可复用：可在多个场景复用                                 │
│ 5. 自动获取主题：通过ThemeProvider获取，无需参数传递        │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 基础控件列表

| 控件 | 功能 | 参数 |
|------|------|------|
| 卡片容器.py | 卡片外轮廓、阴影 | content, height, width |
| 分割线.py | 垂直分割线 | height, enabled |
| 标签文本.py | 文本显示 | text, role, enabled |
| 下拉框.py | 下拉选择 | options, value, on_change |
| 输入框.py | 文本输入 | value, hint_text, on_change |
| 按钮.py | 按钮功能 | text, icon, on_click |
| 头像.py | 用户头像 | image_url, size |
| 主题色块.py | 主题预览 | color, name, on_click |

### 4.3 零件层使用示例


## 五、组件层组合设计

### 5.1 组件层设计原则

```
组件层设计原则：
┌─────────────────────────────────────────────────────────────┐
│ 1. 组合零件：将多个零件组合成完整功能单元                   │
│ 2. 简单交互：处理简单的交互逻辑                             │
│ 3. 不处理业务：复杂业务逻辑由页面层处理                     │
│ 4. 可复用：可在多个页面复用                                 │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 组件列表

| 组件 | 组成 | 功能 |
|------|------|------|
| 标签控件.py | 标签文本 + 下拉框/输入框 | 标签+控件组合 |
| 图标标题.py | 图标 + 标题 + 分割线 + 副标题 | 卡片左侧区域 |
| 导航按钮.py | 按钮 + 选中状态 | 导航按钮 |
| 通用卡片.py | 图标标题 + 控件容器 | 完整卡片 |

# 建筑界面性能优化方案共识

## 一、背景

建筑界面因下拉框选项过多（55个下拉框×20个选项=1100+个DOM节点）导致加载与保存卡顿，需要进行性能优化。

## 二、方案对比汇总

### 2.1 技术方案对比

| 方案 | 核心思路 | 综合评分 |
|------|----------|----------|
| 原懒加载版 | 5模块协调，切换时销毁+创建 | 60.5分 |
| 方案一：分页下拉框 | 保持静态卡片，下拉框分页显示 | 81.5分 |
| 方案二：搜索式输入框 | 输入搜索替代下拉选择 | 82.5分 |
| 方案三：预渲染优化 | 保持现有设计，优化渲染策略 | 84.5分 |
| 方案四：折叠卡片 | 每建筑类型独立卡片，默认折叠 | 87.5分 |
| **折叠版+智能策略** | 即时保存+延迟销毁 | **90分** |

### 2.2 懒加载版 vs 折叠版+智能策略 详细对比

| 维度 | 权重 | 原懒加载版 | 折叠版+智能策略 |
|------|------|------------|-----------------|
| 架构简洁度 | 15% | 60分（5模块） | 90分（1模块） |
| 初始加载性能 | 15% | 70分（~300ms） | 95分（~50ms） |
| 运行时性能 | 20% | 60分（切换卡顿） | 90分（即时响应） |
| 内存管理 | 15% | 80分（立即释放） | 75分（延迟释放） |
| 用户体验 | 20% | 50分（无法对比） | 90分（可多卡片展开） |
| 可维护性 | 15% | 50分（复杂依赖） | 85分（单模块） |
| **加权总分** | 100% | **60.5分** | **87.5分** |

### 2.3 保存/销毁策略对比

| 策略 | 保存时机 | 销毁时机 | 切换响应 | 退出速度 | 内存占用 |
|------|----------|----------|----------|----------|----------|
| A. 即时保存+不销毁 | 值变化时 | 不销毁 | ~0ms | 快 | 高 |
| B. 即时保存+收起销毁 | 值变化时 | 收起时 | ~0ms | 快 | 低 |
| C. 即时保存+导航切换销毁 | 值变化时 | 切换界面时 | ~100ms | 快 | 中 |
| **D. 即时保存+延迟销毁** | 值变化时 | 收起后30秒 | ~0ms | 快 | 中 |

### 2.4 布局风格对比

| 方案 | 代码复用 | 风格一致性 | 维护成本 |
|------|----------|------------|----------|
| 自定义折叠卡片 | 30% | 70% | 高 |
| **基于通用卡片的折叠卡片** | 80% | 100% | 低 |

## 三、共识结论

### 3.1 最优方案

**折叠版 + 即时保存 + 延迟销毁 + 通用卡片布局**

### 3.2 核心特性

1. **延迟加载**：初始只渲染卡片头部+副标题+加载标识，点击后创建控件
2. **即时保存**：值变化时立即保存，避免退出时批量操作
3. **延迟销毁**：卸载控件后30秒销毁，释放内存
4. **通用布局**：基于现有 `IconTitle` + `CardContainer` 组件，保持风格一致

### 3.3 交互设计

**初始状态（未加载控件）**：
```
┌─────────────────────────────────────────────────────────────┐
│  🏠 主帅主城 | 设置主帅主城建筑等级 [▶]                      │
└─────────────────────────────────────────────────────────────┘
```

**点击后（加载控件）**：
```
┌─────────────────────────────────────────────────────────────┐
│  🏠 主帅主城 | 城市▼ 兵工▼ 陆军▼ 空军▼ 商业▼ 补给▼          │
│             | 内塔▼ 村庄▼ 资源▼ 军工▼ 港口▼ 外塔▼          │
└─────────────────────────────────────────────────────────────┘
```

**交互流程**：
1. 初始状态：右侧显示副标题 + 加载标识（▶）
2. 点击右侧区域：副标题消失，加载/显示控件
3. 值变化时：立即触发 `on_save` 回调保存
4. 卸载控件后：30秒后自动销毁释放内存

### 3.3 性能指标

| 指标 | 目标值 |
|------|--------|
| 初始加载时间 | ≤100ms |
| 展开卡片响应时间 | ≤100ms |
| 收起卡片响应时间 | ≤50ms |
| 切换界面响应时间 | ≤50ms |
| 退出应用响应时间 | ≤50ms |

### 3.4 实现要点

1. 创建 `折叠卡片V2.py` 组件模块
2. 复用 `UniversalCard` 组件实现展开状态
3. 实现值变化即时保存回调
4. 实现收起后30秒延迟销毁逻辑
5. 更新 `建筑界面_折叠版.py` 使用新组件

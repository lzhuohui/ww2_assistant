# 配置管理与界面架构实现说明

## 1. 配置管理架构

### 1.1 核心组件

| 组件 | 职责 | 文件路径 |
|------|------|----------|
| ConfigService | 配置管理业务逻辑 | 前端/新界面_v2/业务层/服务/配置服务.py |
| ConfigRepository | 配置文件读写操作 | 前端/新界面_v2/数据层/仓库/配置仓库.py |
| ExportRepository | 游戏配置导出 | 前端/新界面_v2/数据层/仓库/导出仓库.py |

### 1.2 配置文件结构

```
前端/配置/
├── 默认配置.json      # 默认配置文件（只读）
└── 用户配置.json      # 用户当前配置文件（读写）
```

配置数据结构：
```json
{
  "card_id.config_key": "value",
  "hangup_mode.挂机模式": "全自动",
  "account_01.名称": "统帅1"
}
```

## 2. 配置管理流程

### 2.1 启动时加载

1. **新用户**：
   - 用户配置文件不存在
   - 读取默认配置文件
   - 复制默认配置到用户配置文件
   - 加载到内存缓存

2. **老用户**：
   - 用户配置文件存在
   - 直接读取用户配置文件
   - 加载到内存缓存

### 2.2 运行时保存

1. **用户修改配置**：
   - 调用 `ConfigService.set_value()`
   - 更新内存缓存
   - 自动保存到用户配置文件

### 2.3 恢复默认

1. **用户点击恢复默认**：
   - 调用 `ConfigService.reset_to_default()`
   - 复制默认配置到用户配置文件
   - 重新加载到内存缓存

## 3. 功能界面实现

### 3.1 系统界面

- **卡片类型**：折叠卡片
- **配置项**：
  - 挂机模式：全自动/半自动
  - 指令速度：100-500毫秒
  - 尝试次数：10-30次
  - 清缓限量：1.0-5.0M
- **加载逻辑**：创建卡片时从配置服务获取已保存的值

### 3.2 策略界面

- **卡片类型**：折叠卡片
- **配置项**：
  - 速建限级：05-15
  - 速建类型：城资建筑/城市建筑/资源建筑
  - 速产限级：05-15
  - 速产类型：平衡资源/战时经济/钢铁熔炉/橡胶采集/石油开采
  - 保留点数：30-240
- **加载逻辑**：创建卡片时从配置服务获取已保存的值

### 3.3 账号界面

- **卡片类型**：折叠卡片
- **配置项**：
  - 类型：主帅/副帅
  - 平台：Tap/九游/Fan/小7/Vivo/Opop
  - 名称：统帅名称
  - 账号：游戏账号
  - 密码：游戏密码
  - 开关：参与挂机状态
- **加载逻辑**：创建卡片时从配置服务获取已保存的账号信息
- **参与状态计算**：三个输入框都不空+左侧开关打开=参与挂机

## 4. 技术实现要点

### 4.1 简化的配置服务

- 移除了单例模式
- 移除了回调机制
- 移除了默认配置检查逻辑
- 代码行数减少约50%

### 4.2 简单的文件读写

- 默认配置文件：只读，供恢复默认使用
- 用户配置文件：读写，记录用户所有修改
- 启动时自动加载用户配置

### 4.3 内存缓存机制

- 配置文件很小（约3KB），一次性加载不会影响性能
- 内存缓存确保界面响应快速
- 每次修改自动保存到文件

## 6. 界面架构设计

### 6.1 主界面布局结构

```
主窗口 (Page)
└── 外层容器 (padding=USER_SPACING)
    └── 主行容器 (expand=True)
        ├── 导航面板 (固定宽度)
        │   ├── 用户信息卡片
        │   ├── 导航按钮列表
        │   └── 功能按钮列表
        ├── 垂直分割线
        └── 内容区域 (expand=True)
            ├── 页面标题栏
            ├── 分割线
            └── 卡片列表
```

### 6.2 主界面控制逻辑

1. **导航切换**：
   - 点击导航按钮切换功能界面
   - 每次切换前销毁当前加载的卡片
   - 更新导航按钮选中状态
   - 发布页面切换事件

2. **卡片管理**：
   - 每个功能界面包含多个折叠卡片
   - 卡片支持展开/折叠状态
   - 卡片左侧开关控制功能启用状态
   - 卡片内容动态加载

3. **响应式布局**：
   - 导航面板固定宽度
   - 内容区域自适应剩余空间
   - 卡片布局支持多列排列

## 7. 卡片组管理器设计共识

### 7.1 设计目标

1. **统一操作风格**：所有卡片的展开/折叠行为一致
2. **简化界面逻辑**：界面不需要关心卡片切换细节
3. **预留接口**：支持不同控件类型的特殊需求（如色块、主题、调色板等）
4. **易于扩展**：可以轻松添加新的控件类型

### 7.4 各界面原有功能逻辑

#### 系统界面
- 卡片切换时：销毁其他卡片的控件
- 开关状态：保存和加载
- 控件值：保存和加载

#### 策略界面
- 卡片切换时：销毁其他卡片的控件
- 开关状态：保存和加载
- 控件值：保存和加载

#### 账号界面
- 卡片切换时：不销毁控件，只卸载下拉框选项
- 开关状态：保存和加载
- 控件值：保存和加载
- 参与状态计算：三个输入框都不空+左侧开关打开=参与挂机
- 副标题动态更新：根据开关状态和输入内容更新

#### 个性化界面
- 卡片切换时：不销毁控件
- 色块点击：重新创建色块组件更新选中状态
- 主题/调色板/风格切换：更新配置并重新创建色块组件

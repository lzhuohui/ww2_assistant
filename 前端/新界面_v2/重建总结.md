# 新界面_v2 重建总结

## 一、设计思路

### 1. 四层架构设计

```
入口层 → 表示层 → 业务层 → 数据层 → 核心
```

| 层级 | 职责 | 依赖方向 |
|------|------|----------|
| 入口层 | 应用入口，组装各层模块 | 依赖所有层 |
| 表示层 | 界面和组件，纯UI展示 | 依赖业务层和核心层 |
| 业务层 | 服务和事件，业务逻辑 | 依赖数据层和核心层 |
| 数据层 | 仓库和模型，数据存储 | 依赖核心层 |
| 核心层 | 配置、常量、工具 | 无依赖 |

### 2. 模块隔离原则

- **禁止往前覆盖**：不修改被调用方的用户指定变量
- **通过参数传递**：调用方通过函数参数传递配置
- **公开接口访问**：只通过公开接口访问模块

### 3. 用户变量保护

```python
# *** 用户指定变量 - AI不得修改 ***
USER_XXX = "默认值"
# *********************************
```

- 每个模块必须包含用户指定变量区域
- AI不得修改此区域的任何内容
- 优先级：函数参数 > 用户指定变量 > 配置文件 > 代码默认值

## 二、关键技术方案

### 1. 折叠卡片组件

**设计要点**：
- 固定高度卡片，右侧内容切换（副标题↔控件）
- 左侧：图标+标题+分割线，点击切换开关
- 右侧：副标题（展开前）↔ 控件（展开后）

**对齐方案**：
- 展开前：副标题以分割线右侧为基准，左侧居中
- 展开后：控件以卡片右端为基准，右侧居中

```python
副标题容器 = ft.Container(
    alignment=ft.Alignment(0, 0.5),  # 左对齐，垂直居中
)
控件容器 = ft.Container(
    alignment=ft.Alignment(1, 0.5),  # 右对齐，垂直居中
)
```

### 2. 主界面布局对齐

**精确对齐公式**：
```python
top = USER_SPACING + USER_PADDING
```

| 位置 | 计算方式 |
|------|----------|
| 左侧用户卡片内容顶部 | USER_SPACING（导航面板外边距） + USER_PADDING（卡片内边距） |
| 右侧标题栏顶部 | USER_SPACING + USER_PADDING |

### 3. 单例模式优化

```python
class 服务类:
    _instance = None
    
    @classmethod
    def 重置实例:
        """重置单例实例（用于测试或内存释放）"""
        cls._instance = None
    
    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance
```

## 三、经验总结

### 1. Flet布局经验

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 嵌套Container布局异常 | 多层expand=True冲突 | 只在最外层设置expand |
| 内容不显示 | Container缺少expand | 添加expand=True |
| 对齐不生效 | Stack内元素需单独设置alignment | 为每个子元素设置alignment |

### 2. 动画性能优化

```python
# 推荐动画时长
USER_ANIMATION_DURATION = 150  # 毫秒

# 使用Animation对象
animate=ft.Animation(USER_ANIMATION_DURATION, ft.AnimationCurve.EASE_OUT)
```

### 3. 代码规范检查

- 定期运行规范合规检查
- 移除未使用的导入
- 使用最新API（如Button替代ElevatedButton）
- 变量命名避免冗余（如USER_USER_INFO_HEIGHT → USER_INFO_HEIGHT）

## 四、教训总结

### 1. 布局问题排查

**问题**：主界面右侧内容区域顶部间距不正确

**排查过程**：
1. 检查系统界面返回的Container是否设置expand
2. 检查主界面内容容器的嵌套结构
3. 检查多层Container的expand设置冲突

**解决方案**：
- 系统界面返回的Container添加expand=True
- 主界面使用Column包装右侧内容
- 避免多层嵌套的expand=True

### 2. 用户变量冲突

**问题**：多个模块定义相同的用户变量

**解决方案**：
- 每个模块只定义自己需要的用户变量
- 从被调用模块导入默认值，禁止硬编码
- 主界面作为入口层，定义全局布局变量

### 3. API兼容性

**问题**：使用已弃用的API

**解决方案**：
- 定期检查Flet版本更新
- 使用最新API替代已弃用API
- 关注DeprecationWarning警告

### 2. 组件复用检查

- 新建模块前，先搜索是否有可复用组件
- 功能相似度 > 80% 可直接复用
- 功能相似度 50-80% 可扩展后复用
- 功能相似度 < 50% 新建模块

### 3. 调试代码格式

```python

    配置 = 界面配置()

```

- 禁止添加初始化代码
- 禁止设置页面属性
- 禁止使用非默认参数

## 六、折叠卡片设计模式

### 1. 双模式支持

折叠卡片支持**配置模式**和**只读模式**：

```python
# 配置模式（默认）
折叠卡片.创建(
    标题="速建限级",
    图标="APARTMENT",
    副标题="设置建筑等级",
    控件=[下拉框1, 下拉框2],  # 点击展开显示
)

# 只读模式
折叠卡片.创建(
    标题="版本信息",
    图标="INFO",
    只读模式=True,  # 无开关、无展开动画
    控件=[信息行1, 信息行2],  # 直接显示
)
```

### 2. 模式对比

| 特性 | 配置模式 | 只读模式 |
|------|----------|----------|
| **开关** | ✅ 左侧点击切换 | ❌ 无开关 |
| **展开动画** | ✅ 点击展开显示控件 | ❌ 无动画 |
| **右侧内容** | 副标题 → 控件切换 | 控件直接显示 |
| **适用场景** | 策略、任务、建筑等配置 | 关于、打扫等展示 |

### 3. 控件参数设计

折叠卡片支持两种方式传入控件：

```python
# 方式A：直接传入控件（推荐）
控件列表 = [下拉框1, 输入框1]
卡片 = 折叠卡片.创建(控件=控件列表, ...)

# 方式B：传入配置字典（兼容）
控件配置 = [{"type": "dropdown", "config_key": "xxx", ...}]
卡片 = 折叠卡片.创建(控件配置=控件配置, ...)
```

## 七、导航栏功能按钮设计

### 1. 设计思路

将导航栏底部的功能按钮（如"导出配置"）与导航按钮使用相同样式，保持视觉统一。

### 2. 布局结构

```
导航栏布局：
├── 用户信息卡片
├── 导航按钮列表
│   ├── 系统
│   ├── 策略
│   ├── ...
│   └── 关于
├── 功能按钮列表
│   ├── 导出配置
│   ├── 恢复默认（未来扩展）
│   └── 配置方案（未来扩展）
```

### 3. 未来扩展方向

| 功能按钮 | 说明 |
|----------|------|
| **导出配置** | 保存当前配置到文件 |
| **恢复默认** | 重置所有配置为默认值 |
| **配置方案** | 多套配置切换，适应不同游戏阶段 |

### 4. 实现要点

- 功能按钮使用与导航按钮相同的样式
- 功能按钮点击后执行操作，不切换页面
- 功能按钮列表可动态扩展

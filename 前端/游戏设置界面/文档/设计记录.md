### 共识 #004：三种销毁策略

| 项目 | 内容 |
|------|------|
| **达成时间** | 2026-03-25 |
| **问题描述** | 不同配置区对卡片切换时的处理需求不同 |
| **讨论方案** | A：统一销毁策略<br>B：支持多种策略 |
| **最终决策** | 支持三种销毁策略：none/unload_options/destroy_controls |
| **决策原因** | 满足不同配置区的特殊需求，保持灵活性 |

| 策略 | 适用场景 | 行为 |
|------|----------|------|
| `none` | 账号配置区、个性化配置区 | 不销毁控件 |
| `unload_options` | 系统配置区、策略配置区 | 只卸载下拉框选项 |
| `destroy_controls` | 预留 | 销毁所有控件 |

### 共识 #007：采用Win11 Fluent Design设计规范

| 项目 | 内容 |
|------|------|
| **达成时间** | 2026-03-25 |
| **问题描述** | 项目需要统一的UI设计规范 |
| **讨论方案** | A：自定义设计风格<br>B：遵循Win11 Fluent Design |
| **最终决策** | 采用Win11 Fluent Design设计规范 |
| **决策原因** | 与Windows系统风格一致，用户体验熟悉，设计规范完整 |

**Fluent Design五大核心要素**：

| 要素 | 说明 | 应用场景 |
|------|------|----------|
| Light (光照) | 高亮、聚焦、选中效果 | 悬停、焦点状态 |
| Depth (深度) | 层级阴影、Z-Index | 卡片、弹出菜单、对话框 |
| Motion (运动) | 动画过渡效果 | 状态切换、页面切换 |
| Material (材质) | Acrylic、Mica材质 | 窗口背景、弹出菜单 |
| Scale (缩放) | 响应式布局 | 不同屏幕尺寸适配 |

**详细规范文档**：[Win11设计规范.md](./Win11设计规范.md)

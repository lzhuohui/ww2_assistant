# 界面性能优化经验总结

## 项目背景

### 问题描述
建筑设置页面存在严重的性能问题：
- 进入页面慢：需要创建5个卡片，55个下拉框，约1100个选项
- 退出页面慢：需要销毁所有控件，保存所有数据
- 操作后退出更慢：操作下拉框后退出时间长达3-4秒

### 优化目标
- 提升进入页面速度
- 提升退出页面速度
- 降低内存占用
- 保持用户体验流畅

---

## 优化方案

### 方案演进

| 阶段 | 方案 | 结果 |
|------|------|------|
| 1 | 下拉框懒加载 | 效果有限，实现复杂 |
| 2 | 配置延迟保存 | 效果有限 |
| 3 | **卡片级别懒加载** | **效果显著，最终方案** |

### 最终方案：卡片级别懒加载

```
核心思路：
1. 进入页面：只加载1个卡片（默认加载"主帅主城"）
2. 其他卡片：显示"📁 点击加载数据"
3. 切换卡片：保存上一个 + 销毁上一个 + 加载当前
4. 退出页面：只销毁1个卡片
```

## 技术实现

### 1. 懒加载卡片管理器（单例模式）

```python
class LazyCardManager:
    """懒加载卡片管理器（单例）"""
    
    _instance = None
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance._initialized = False
        return cls._instance
    
    def switch_to(self, card_name: str):
        """切换到指定卡片"""
        # 保存当前卡片数据
        if self.current_card_name and self.config_manager:
            self.config_manager.save_all()
        
        # 销毁当前卡片
        if self.current_card_name and self.current_card_name in self.cards:
            self.cards[self.current_card_name].destroy()
        
        # 加载新卡片
        if card_name in self.cards:
            self.cards[card_name].load()
            self.current_card_name = card_name
```

### 2. 懒加载通用卡片

```python
class LazyUniversalCard:
    """懒加载通用卡片"""
    
    def create(self) -> ft.Container:
        """创建卡片容器"""
        if self.is_default:
            # 默认加载：直接创建已加载内容
            self.container = self._create_loaded_container()
            self.is_loaded = True
        else:
            # 懒加载：显示"点击加载"
            self.container = self._create_unloaded_container()
            self.container.on_click = self._on_click
        
        return self.container
    
    def load(self):
        """加载卡片内容"""
        if self.is_loaded:
            return
        self.container.content = self._create_loaded_content()
        self.is_loaded = True
    
    def destroy(self):
        """销毁卡片内容"""
        if not self.is_loaded:
            return
        self.container.content = self._create_unloaded_content()
        self.is_loaded = False
```

### 3. 状态同步

```python
def _sync_controls_state(self, enabled: bool, controls: list):
    """同步控件的状态"""
    for control in controls:
        control.opacity = 1.0 if enabled else 0.4
        if hasattr(control, 'set_state'):
            control.set_state(enabled)
```

## 经验教训

### 1. 问题定位要准确

**错误做法**：一开始就尝试优化下拉框级别
**正确做法**：先分析性能瓶颈，定位到卡片级别

### 2. 方案选择要合理

| 方案 | 复杂度 | 效果 | 推荐度 |
|------|--------|------|--------|
| 下拉框懒加载 | 高 | 有限 | ⭐⭐ |
| 配置延迟保存 | 低 | 有限 | ⭐⭐⭐ |
| 卡片懒加载 | 中 | 显著 | ⭐⭐⭐⭐⭐ |

### 3. 保持代码一致性

- 懒加载卡片布局与原始卡片保持一致
- 使用相同的组件（IconTitleV2、CardContainer）
- 保持相同的状态切换功能

### 4. 安全第一

- 先Git备份再整理目录
- 保留原始版本，新增优化版本
- 测试通过后再删除临时文件

## 总结

卡片级别懒加载是最有效的优化方案，它解决了核心性能问题，同时保持了代码的简洁性和可维护性。关键经验：

1. **先分析后优化**：定位瓶颈再动手
2. **选择合适方案**：卡片级别比下拉框级别更有效
3. **保持一致性**：布局和功能与原始版本一致
4. **安全第一**：Git备份保护代码安全

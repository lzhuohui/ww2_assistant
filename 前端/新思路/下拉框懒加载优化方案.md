# 下拉框懒加载优化方案

## 讨论日期
2026-03-13

---

## 一、问题背景

### 1.1 当前问题
- 建筑设置页面有5个卡片，55个下拉框，约1100个选项
- 进入页面时，创建所有选项，导致页面加载慢
- 退出页面时，销毁所有选项，导致页面退出慢
- 退出页面时，保存所有数据，导致退出慢

### 1.2 性能问题
- **进入页面**：创建1100个选项（慢）
- **退出页面**：销毁1100个选项 + 保存所有数据（慢）

---

## 二、解决方案

### 2.1 核心思路
**懒加载 + 精准销毁 + 延迟保存**

### 2.2 关键技术点

#### 1. 只加载当前页面的下拉框按钮
- 进入页面时，只创建下拉框按钮
- 每个按钮显示默认值（比如"17"）
- 不创建选项列表
- 界面美观，用户可以看到当前值

#### 2. 懒加载选项列表
- 第一次展开下拉框时，才创建选项列表
- 标记为"已创建"（后台标记）

#### 3. 精准销毁
- 只销毁上一个已创建的下拉框
- 不销毁未创建的下拉框
- 减少不必要的操作

#### 4. 延迟保存
- 选择选项时，只更新内存中的值
- 点到下一个下拉框时，保存上一个下拉框的值到配置文件
- 然后销毁上一个下拉框的选项

---

## 三、实施步骤

### 3.1 创建下拉框时（初始状态）
- 只加载当前页面的下拉框按钮
- 每个按钮显示默认值
- 不创建选项列表

### 3.2 第一次展开时（懒加载）
- 检查上一个已创建的下拉框
- 如果有，保存上一个下拉框的值到配置文件
- 然后销毁上一个下拉框的选项
- 创建当前下拉框的选项列表
- 标记为"已创建"（后台标记）

### 3.3 选择选项时（更新内存）
- 更新选中值
- 更新内存中的配置
- 不保存到文件（延迟保存）
- 收起下拉列表

### 3.4 点到下一个下拉框时（保存-销毁）
- 保存上一个下拉框的值到配置文件
- 销毁上一个下拉框的选项
- 创建当前下拉框的选项

### 3.5 退出页面时（保存所有）
- 保存所有已修改的值到配置文件
- 销毁所有下拉框的选项

---

## 四、性能对比

### 4.1 当前方案（PopupMenuButton）
- **进入页面**：创建1100个选项（慢）
- **选择选项**：正常
- **退出页面**：保存所有数据 + 销毁1100个选项（慢）

### 4.2 最优方案（自定义下拉框 + 懒加载 + 精准销毁 + 延迟保存）
- **进入页面**：只创建按钮，不创建选项列表（快）
- **点击下拉框**：销毁上一个，创建当前（快）
- **选择选项**：更新内存（快）
- **点到下一个下拉框**：保存上一个，销毁上一个（快）
- **内存占用**：始终只有一个下拉框的选项（最低）
- **退出页面**：只保存一次（快）

## 五、关键技术实现

### 5.1 下拉框管理器（全局）

```python
class DropdownManager:
    """下拉框管理器（全局）"""
    
    last_dropdown = None  # 上一个已创建的下拉框
    last_dropdown_value = None  # 上一个下拉框的值
    
    @classmethod
    def open_dropdown(cls, dropdown):
        """打开下拉框"""
        # 保存上一个下拉框的值到配置文件
        if cls.last_dropdown and cls.last_dropdown != dropdown:
            cls.last_dropdown.save_to_config()
            # 销毁上一个下拉框的选项
            cls.last_dropdown.destroy_options()
        
        # 创建当前下拉框的选项
        if not dropdown.options_created:
            dropdown.create_options()
        
        # 记录当前下拉框
        cls.last_dropdown = dropdown
```

### 5.2 自定义下拉框

```python
class LazyDropDown:
    """自定义下拉框（懒加载版本）"""
    
    def __init__(self, config, options, value, width, height, on_change):
        self.config = config
        self.options = options
        self.value = value
        self.width = width
        self.height = height
        self.on_change = on_change
        
        # 状态标志
        self.dropdown_opened = False
        self.options_created = False
        
        # 创建UI
        self.create_ui()
    
    def create_ui(self):
        """创建UI"""
        # 创建按钮（显示默认值）
        self.button = self.create_button()
        
        # 创建下拉列表容器（初始为空）
        self.dropdown_list = ft.ListView(
            visible=False,
            height=200,  # 固定高度，支持滚动
            item_extent=30,  # 每个选项的高度
        )
        
        # 创建Stack管理层级
        self.control = ft.Stack([
            self.button,
            self.dropdown_list,
        ])
    
    def create_button(self):
        """创建按钮"""
        # 显示默认值
        self.selected_text = ft.Text(
            self.value,  # 显示默认值（比如"17"）
            size=14,
            color=theme_colors.get("text_primary"),
        )
        
        # 创建按钮容器
        button_container = ft.Container(
            content=ft.Row([
                self.selected_text,  # 显示默认值
                ft.Icon(ft.Icons.KEYBOARD_ARROW_DOWN, size=18),
            ]),
            width=self.width,
            height=self.height,
            border_radius=6,
            bgcolor=theme_colors.get("bg_secondary"),
            border=ft.Border.all(1, theme_colors.get("border")),
            padding=ft.Padding(left=12, right=8, top=0, bottom=0),
            on_click=self.toggle_dropdown,
        )
        
        return button_container
    
    def create_options(self):
        """创建选项（懒加载）"""
        for option in self.options:
            item = ft.Container(
                content=ft.Text(option, size=14),
                on_click=lambda e, o=option: self.select_option(o),
            )
            self.dropdown_list.controls.append(item)
        
        self.options_created = True
        self.dropdown_list.update()
    
    def toggle_dropdown(self):
        """切换下拉列表"""
        if not self.dropdown_opened:
            # 展开时，通知管理器
            DropdownManager.open_dropdown(self)
        
        self.dropdown_opened = not self.dropdown_opened
        self.update_visibility()
    
    def select_option(self, option):
        """选择选项"""
        # 更新选中值
        self.value = option
        self.selected_text.value = option
        self.selected_text.update()
        
        # 更新内存中的配置（不保存到文件）
        config_manager.set_value(self.config_key, option)
        
        # 收起下拉列表
        self.dropdown_opened = False
        self.update_visibility()
    
    def save_to_config(self):
        """保存到配置文件"""
        config_manager.save_to_file()
    
    def destroy_options(self):
        """销毁选项"""
        if self.options_created:
            # 清空下拉列表
            self.dropdown_list.controls.clear()
            self.dropdown_list.update()
            # 标记为未创建
            self.options_created = False
```

## 六、总结

### 6.1 核心优势
1. **进入页面快**：只创建按钮，不创建选项列表
2. **界面美观**：显示默认值
3. **使用时流畅**：懒加载选项列表
4. **减少文件写入次数**：延迟保存
5. **内存占用最低**：始终只有一个下拉框的选项
6. **退出页面快**：只保存一次

### 6.2 关键点
1. **只加载当前页面的下拉框按钮**：减少初始加载时间
2. **只销毁上一个已创建的下拉框**：减少不必要的操作
3. **点到下一个下拉框时保存-销毁**：减少文件写入次数，提升性能

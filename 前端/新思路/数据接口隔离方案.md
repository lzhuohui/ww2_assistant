# 数据接口隔离方案

## 方案背景

### 问题描述
在项目开发过程中，出现了调用者模块覆盖被调用模块用户指定数据的问题，违反了"AI被调保护规范"。

### 问题根源
1. 调用者模块通过外部接口获取被调用模块的用户指定数据
2. 调用者模块硬编码默认值，覆盖了被调用模块的默认值
3. 缺乏清晰的数据流转规则

## 方案目标

### 核心目标
确保调用者模块正确继承被调用模块内部的数据，而非通过外部接口获取。

### 具体目标
1. 建立清晰的数据流转规则
2. 增强数据流转的可控性
3. 提高系统的整体稳定性
4. 避免未来再次出现类似问题

## 实施案例

### 案例：下拉框宽度问题

#### 当前问题
- 调用者模块（控件工厂）通过外部接口获取宽度参数
- 调用者模块（开关下拉卡片）硬编码默认宽度
- 被调用模块（自定义下拉框）有用户指定的默认宽度50

#### 解决方案
1. 断开控件工厂的外部宽度参数接口
2. 移除开关下拉卡片中的硬编码默认宽度
3. 让自定义下拉框使用自己的默认宽度50
4. 如果用户指定了宽度，通过配置文件传递

#### 实施步骤
1. 修改控件工厂._create_dropdown方法，不传递width参数
2. 修改控件工厂._create_switch_dropdown_controls方法，不传递width参数
3. 修改开关下拉卡片.create方法，不设置默认宽度
4. 测试所有下拉框控件

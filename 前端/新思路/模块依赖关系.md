# 模块依赖关系图

## 零件层（基础模块，禁止修改）

### 核心零件
- **自定义下拉框.py** - 独立功能模块，提供下拉选择功能
- **帮助标签.py** - 独立功能模块，提供帮助提示功能
- **分割线.py** - 独立功能模块，提供分割线功能
- **卡片容器.py** - 独立功能模块，提供卡片容器功能

### 组合零件（可修改）
- **图标标题.py** - 组合模块，可修改（调用帮助标签、分割线）
- **标签下拉框.py** - 组合模块，可修改（调用自定义下拉框）
- **标签输入框.py** - 组合模块，可修改

## 组件层（装配模块，可修改）

- **通用卡片.py** - 装配模块，可修改（调用图标标题、卡片容器）
- **基础设置卡片.py** - 装配模块，可修改（调用通用卡片、标签下拉框、标签输入框）
- **开关下拉卡片.py** - 装配模块，可修改（调用通用卡片、标签下拉框、标签输入框）

## 修改权限规则

### 规则1：零件层核心模块禁止修改
- 自定义下拉框.py
- 帮助标签.py
- 分割线.py
- 卡片容器.py

### 规则2：组合零件只调用不覆盖
- 图标标题.py - 只调用帮助标签、分割线，不重新定义它们的默认值
- 标签下拉框.py - 只调用自定义下拉框，不重新定义它的默认值

### 规则3：组件层只装配不干预
- 通用卡片.py - 只调用图标标题，不干预其内部布局
- 基础设置卡片.py - 只调用通用卡片，不干预其内部逻辑

## 违规检测清单

在修改任何代码前，必须检查：

1. [ ] 我是否正在修改零件层核心模块？（如果是，立即停止）
2. [ ] 我是否在组合零件中重新定义了被调用模块的默认值？（如果是，立即停止）
3. [ ] 我是否在组件层中干预了被调用模块的内部布局？（如果是，立即停止）
4. [ ] 我是否查看了当前模块的所有import语句？（必须查看）
5. [ ] 我是否确认了所有被调用模块的修改权限？（必须确认）

## 常见违规模式

### 违规模式1：重新定义默认值
```python
# ❌ 错误：在标签下拉框中重新定义默认宽度
default_width = ui_config.get("dropdown_width", 120)
current_width = width if width is not None else default_width

# ✅ 正确：直接传递参数，让被调用模块使用自己的默认值
dropdown_control = CustomDropDown.create(
    config=config,
    width=width,  # 直接传递，不重新赋值
)
```

### 违规模式2：干预内部布局
```python
# ❌ 错误：在通用卡片中干预图标标题的布局
left_container = ft.Container(
    content=icon_title,
    top=0,
    bottom=0,
    alignment=ft.Alignment(0, 0),  # 强制居中，干预内部布局
)

# ✅ 正确：简单放置，不干预内部布局
left_container = ft.Container(
    content=icon_title,
    left=card_padding,
    top=10,
)
```

```python
# 在标签下拉框.py中修改自定义下拉框.py

# ✅ 正确：只通过接口调用，不修改被调用模块
dropdown_control = CustomDropDown.create(...)
```

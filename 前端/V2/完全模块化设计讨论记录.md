## 一、背景

在V2重构过程中，发现项目存在以下违反模块化设计原则的问题：

| 问题类型 | 具体表现 | 违反原则 |
|---------|---------|---------|
| 重复代码 | 7个功能界面代码结构完全相同 | DRY原则 |
| 模块化不完整 | 功能界面硬编码CARDS_CONFIG | 数据归属原则 |
| 模块不独立 | enabled状态重复获取 | 模块隔离原则 |
| 数据多次覆盖 | 参数传递冗余 | 单一数据源原则 |
| 被调模块数据未继承 | 功能界面未使用配置服务接口 | 依赖倒置原则 |
| 测试逻辑违规 | 部分测试代码返回空数据 | 测试约束原则 |

## 二、整改方向讨论

### 2.1 整改顺序讨论

**问题**：要实现完全模块化，是从第5层开始实现还是第1层往下处理？

**两种方案对比**：

| 方案 | 方向 | 优点 | 缺点 |
|------|------|------|------|
| A | 自底向上（层级5→层级1） | 基础模块先稳定，上层模块可以依赖稳定的基础 | 当前层级5已基本正确，可能做无用功 |
| B | 自顶向下（层级2→层级5） | 直接解决核心问题，修改范围可控 | 需要同时修改多层接口 |

**结论**：采用方案B（从第2层往下处理）

**理由**：
1. 问题根源在层级2（功能界面硬编码CARDS_CONFIG）
2. 层级5已经基本正确，无需大改
3. 符合"修改影响范围控制"原则
4. 可以逐层验证修改效果

### 3.3 下拉框模块四层架构实现

```python
# -*- coding: utf-8 -*-

"""
模块名称：下拉框.py
模块功能：下拉框组件 - 完全模块化设计

四层架构：
第1层：组件独立功能 - 数据获取、懒加载、风格统一
第2层：上层调用 - 只负责布局逻辑
第3层：用户指定变量覆盖 - 可覆盖默认值
第4层：公开接口 - 供上层对接
"""

import flet as ft
from typing import List, Dict, Callable, Any, Optional

# ============================================
# 第1层：组件独立功能 - 默认配置（fallback）
# ============================================

DEFAULT_WIDTH = 120
DEFAULT_HEIGHT = 30

DEFAULT_THEME = {
    "text_primary": "#000000",
    "text_secondary": "#666666",
    "text_disabled": "#999999",
    "bg_primary": "#FFFFFF",
    "bg_secondary": "#F5F5F5",
    "border": "#CCCCCC",
    "accent": "#0078D4"
}

# ============================================
# 第4层：公开接口
# ============================================

class Dropdown:
    """
    下拉框组件 - 完全模块化设计
    
    第1层职责（组件独立功能）：
    - 从配置服务获取选项列表
    - 从配置服务获取当前值
    - 从配置服务获取默认值
    - 从配置服务获取UI配置（宽度、高度）
    - 懒加载菜单
    - 自动保存到配置
    
    第2层职责（上层调用）：
    - 上层只传递section和control_id
    - 上层只负责布局逻辑
    
    第3层职责（用户指定变量覆盖）：
    - theme_colors可覆盖默认主题
    - width/height可覆盖默认尺寸（特殊场景）
    - enabled可覆盖默认状态（特殊场景）
    
    第4层职责（公开接口）：
    - create() 创建下拉框
    - get_width/get_height() 获取尺寸
    - get_options() 获取选项列表
    - get_current_value() 获取当前值
    - get_default_value() 获取默认值
    - reset_to_default() 重置为默认值
    - destroy_all() 销毁所有下拉框
    """
    
    _config_service = None
    
    # ============================================
    # 第4层：公开接口 - 静态方法
    # ============================================
    
    @classmethod
    def set_config_service(cls, config_service):
        """设置配置服务实例"""
        cls._config_service = config_service
    
    @staticmethod
    def get_width() -> int:
        """获取下拉框宽度（从配置服务获取，fallback到默认值）"""
        if Dropdown._config_service:
            return Dropdown._config_service.get_ui_config("下拉框", "宽度", DEFAULT_WIDTH)
        return DEFAULT_WIDTH
    
    @staticmethod
    def get_height() -> int:
        """获取下拉框高度（从配置服务获取，fallback到默认值）"""
        if Dropdown._config_service:
            return Dropdown._config_service.get_ui_config("下拉框", "高度", DEFAULT_HEIGHT)
        return DEFAULT_HEIGHT
    
    # ============================================
    # 第4层：公开接口 - 实例方法
    # ============================================
    
    def __init__(self, page: ft.Page = None, config_service=None):
        self._page = page
        self._config_service = config_service or Dropdown._config_service
        self._dropdowns: Dict[str, ft.Container] = {}
    
    def create(
        self,
        section: str = "",
        control_id: str = "",
        on_change: Callable[[str, str, Dict[str, str]], None] = None,
        theme_colors: Dict[str, str] = None,
        width: int = None,
        height: int = None,
        enabled: bool = True,
    ) -> ft.Container:
        """
        创建下拉框
        
        第1层：数据由模块自己获取
        - 选项列表：_get_options()
        - 当前值：_get_current_value()
        - 默认值：_get_default_value()
        - 宽度/高度：get_width()/get_height()
        
        第2层：上层只传递定位信息
        - section: 配置节
        - control_id: 控件ID
        
        第3层：用户指定变量可覆盖
        - theme_colors: 覆盖默认主题
        - width/height: 覆盖默认尺寸（特殊场景）
        - enabled: 接收控件启用状态
        """
        # 第1层：数据由模块自己获取
        options = self._get_options(section, control_id)
        current_value = self._get_current_value(section, control_id, options)
        
        # 第1层：UI配置由模块自己获取
        actual_width = width if width is not None else Dropdown.get_width()
        actual_height = height if height is not None else Dropdown.get_height()
        
        # 第3层：用户指定变量覆盖
        if theme_colors is None:
            theme_colors = DEFAULT_THEME
        
        # ... 创建UI组件的代码 ...
        
        return self._build_container(
            options, current_value, enabled,
            actual_width, actual_height, theme_colors,
            section, control_id, on_change
        )
    
    def get_options(self, section: str, control_id: str) -> List[Dict[str, str]]:
        """公开接口：获取选项列表"""
        return self._get_options(section, control_id)
    
    def get_current_value(self, section: str, control_id: str) -> Dict[str, str]:
        """公开接口：获取当前值"""
        options = self._get_options(section, control_id)
        return self._get_current_value(section, control_id, options)
    
    def get_default_value(self, section: str, control_id: str) -> str:
        """公开接口：获取默认值"""
        return self._get_default_value(section, control_id)
    
    def reset_to_default(self, section: str, control_id: str) -> bool:
        """公开接口：重置为默认值"""
        default_value = self._get_default_value(section, control_id)
        if default_value:
            options = self._get_options(section, control_id)
            for opt in options:
                if opt.get("value") == default_value or opt.get("text") == default_value:
                    self._save_value(section, control_id, opt)
                    return True
        return False
    
    def destroy_all(self):
        """销毁所有下拉框菜单"""
        self._dropdowns.clear()
    
    # ============================================
    # 第1层：组件独立功能 - 内部逻辑
    # ============================================
    
    def _get_options(self, section: str, control_id: str) -> List[Dict[str, str]]:
        """第1层：从配置服务获取选项列表"""
        if self._config_service is None:
            return [{"text": "选项A", "value": "a"}, {"text": "选项B", "value": "b"}]
        return self._config_service.get_options(section, control_id)
    
    def _get_current_value(self, section: str, control_id: str, options: List[Dict]) -> Dict[str, str]:
        """第1层：从配置服务获取当前值（优先用户配置，其次默认值）"""
        if self._config_service:
            # 优先从用户配置获取
            value = self._config_service.get_text_value(section, control_id)
            if value and value.get("text"):
                return value
            
            # 没有用户配置时，从界面配置获取默认值
            default_value = self._get_default_value(section, control_id)
            if default_value:
                for opt in options:
                    if opt.get("value") == default_value or opt.get("text") == default_value:
                        return opt
        
        # 最后fallback到第一个选项
        if options:
            return options[0]
        return {"text": "", "value": ""}
    
    def _get_default_value(self, section: str, control_id: str) -> str:
        """第1层：从配置服务获取默认值"""
        if self._config_service:
            return self._config_service.get_control_default(section, control_id)
        return ""
    
    def _save_value(self, section: str, control_id: str, value: Dict[str, str]):
        """第1层：保存值到配置"""
        if self._config_service:
            self._config_service.set_value(section, control_id, value)
    
    def _build_container(self, ...):
        """内部方法：构建UI容器"""
        # ... UI构建代码 ...
        pass
```

### 5.3 需要改进的部分

**功能界面应该：**
1. 从config_service.get_card_info(section)获取title、icon、subtitle
2. 从config_service.get_controls(section)获取控件列表
3. 移除硬编码的CARDS_CONFIG

**正确示例：**
```python
class BuildingPage:
    @staticmethod
    def create(page: ft.Page, config_service, on_change=None, theme_colors=None):
        card_group = CardGroup(page, config_service)
        cards = []
        
        sections = config_service.get_all_sections()
        for section in sections:
            if section.startswith("建筑设置."):
                card_info = config_service.get_card_info(section)
                controls = config_service.get_controls(section)
                
                card = card_group.create(
                    section=section,
                    title=card_info.get("title"),
                    icon=card_info.get("icon"),
                    subtitle=card_info.get("subtitle"),
                    controls_config=controls,
                    theme_colors=theme_colors,
                )
                cards.append(card)
        
        return ft.Column(cards, spacing=USER_CARD_SPACING, scroll=ft.ScrollMode.AUTO, expand=True)
```

### 7.6 已实施的修改

**下拉框.py `_get_current_value()` 方法已修改**：

```python
def _get_current_value(self, section: str, control_id: str, options: List[Dict]) -> Dict[str, str]:
    """
    从配置服务获取当前值
    
    加载逻辑：
    1. 有用户配置值 → 显示用户配置的值
    2. 无用户配置值（APK第一次安装或文件损坏）→ 显示"请选定数据..."
    
    注意：保留原有懒加载功能（PopupMenuButton行为不变）
    """
    if self._config_service:
        value = self._config_service.get_text_value(section, control_id)
        if value and value.get("text") and value.get("text") != "":
            return value
    
    return {"text": "请选定数据...", "value": ""}
```

### 8.2 `_get_theme_colors()` 实现规范

**正确实现**：
```python
@staticmethod
def _get_theme_colors() -> Dict[str, str]:
    """获取主题颜色"""
    if Icon._config_service is None:
        raise RuntimeError(
            "Icon模块未设置config_service，"
            "请先调用 Icon.set_config_service(config_service)"
        )
    return Icon._config_service.get_theme_colors()
```

**错误实现（已废弃）**：
```python
# 错误1：硬编码fallback（冗余，配置仓库已有fallback）

    if Icon._config_service:

    return {"text_primary": "#FFFFFF", ...}  # 冗余！

# 错误2：无检查直接调用（会报AttributeError）
def _get_theme_colors() -> Dict[str, str]:
```

# 配置驱动UI动态扩展共识

## 一、背景

当前V2架构存在以下问题：
- 导航按钮硬编码在主入口
- 功能界面硬编码过滤条件
- 添加新界面/卡片需要修改代码

## 二、目标

用户只修改界面配置.json，实现全链路联动：
1. 导航按钮 <> 功能界面：界面增减
2. 功能界面 <> 卡片控件：卡片增减
3. 卡片控件 <> 控件参数：控件增减（已实现）

## 三、核心原则

| 原则 | 说明 |
|------|------|
| 配置驱动 | 界面配置.json是唯一数据源 |
| 顺序即排序 | 配置文件中的顺序就是显示顺序 |
| 简洁无冗余 | 不添加额外的排序字段 |

## 四、界面配置.json结构

```json
{
  "界面配置": {
    "系统设置.挂机模式": {
      "所属界面": "系统界面",
      "卡片信息": {
        "title": "挂机模式",
        "icon": "POWER_SETTINGS_NEW",
        "subtitle": "全自动:自动挂机...",
        "enabled": true
      },
      "控件列表": [
        {"id": "挂机模式", "type": "dropdown", "label": "模式选择", "options": ["全自动", "半自动"], "default": "全自动"}
      ]
    }
  }
}
```

## 五、字段定义

### 5.1 Section层级

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| Section键名 | string | ✓ | 唯一标识，格式：界面名.卡片名 |
| 所属界面 | string | ✓ | 对应功能界面类名 |

### 5.2 卡片信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | ✓ | 卡片标题 |
| icon | string | ✓ | 图标名称 |
| subtitle | string | ✓ | 副标题说明 |
| enabled | boolean | ✓ | 开关状态 |

### 5.3 控件列表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✓ | 控件标识 |
| type | string | ✓ | 控件类型：dropdown/input |
| label | string | ✓ | 控件标签 |
| options | array | 条件 | 下拉选项（dropdown必填） |
| default | string | ✓ | 默认值 |
| hint | string | 条件 | 输入提示（input可选） |
| password | boolean | 条件 | 密码框（input可选） |

## 六、排序规则

| 层级 | 排序依据 | 实现方式 |
|------|---------|---------|
| 导航按钮 | 界面首次出现顺序 | 遍历section，按"所属界面"首次出现排序 |
| 卡片 | section在配置文件中的顺序 | 保持JSON原始顺序 |
| 控件 | 控件列表中的顺序 | 保持数组原始顺序 |

## 七、配置服务新增接口

| 接口 | 返回值 | 说明 |
|------|--------|------|
| get_interfaces() | List[str] | 按首次出现顺序返回界面列表 |
| get_sections_by_interface(interface) | List[str] | 返回指定界面的section列表（保持顺序） |
| get_interface(section) | str | 返回section所属界面 |

## 八、实现范围

| 文件 | 修改内容 |
|------|---------|
| 界面配置.json | 增加"所属界面"字段 |
| 配置服务.py | 增加三个新接口 |
| 主入口.py | 动态生成导航按钮 |
| 功能界面.py | 使用新接口过滤section |

## 九、预期效果

添加新卡片只需在界面配置.json中添加：
```json
"系统设置.新功能": {
  "所属界面": "系统界面",
  "卡片信息": {...},
  "控件列表": [...]
}
```

系统自动：
1. 在系统界面显示新卡片
2. 卡片按配置顺序排列
3. 控件按配置顺序显示

# 新界面重构实施方案

## 一、项目背景

### 1.1 原有问题

| 问题 | 说明 |
|------|------|
| 架构混乱 | 多种设计模式混用（懒加载、折叠卡片等） |
| 代码质量低 | 模块耦合度高，难以维护和扩展 |
| 性能问题 | 下拉框选项过多导致加载卡顿 |
| 系统性差 | 缺乏统一的设计规范和模块化架构 |

### 1.2 重构目标

| 目标 | 说明 |
|------|------|
| 完全模块化 | 每个模块独立运行、测试、维护 |
| 统一设计模式 | 所有界面采用折叠卡片V2模式 |
| 性能优化 | 延迟加载+即时保存+延迟销毁 |
| 易于扩展 | 新增功能只需添加模块 |

## 二、核心设计共识

### 2.1 折叠卡片V2模式

**交互设计**：
```
初始状态（卡片高度固定）：
┌─────────────────────────────────────────────────────────────┐
│  🏠 主帅主城 | 设置主帅主城建筑等级 [▶]                      │
└─────────────────────────────────────────────────────────────┘

点击后（右侧内容切换，带动画）：
┌─────────────────────────────────────────────────────────────┐
│  🏠 主帅主城 | 城市▼ 兵工▼ 陆军▼ 空军▼ 商业▼ 补给▼          │
│             | 内塔▼ 村庄▼ 资源▼ 军工▼ 港口▼ 外塔▼          │
└─────────────────────────────────────────────────────────────┘
```

**核心特性**：
| 特性 | 实现方式 |
|------|----------|
| 卡片高度固定 | 使用 `USER_HEIGHT`，无展开/收起效果 |
| 右侧内容切换 | 副标题 ↔ 控件，通过 `opacity` 动画切换 |
| 延迟加载控件 | 点击右侧区域时才创建下拉框 |
| 延迟创建菜单项 | 打开下拉菜单时才创建选项 |
| 即时保存 | 值变化时立即触发 `on_save` 回调 |
| 延迟销毁 | 卸载控件后30秒自动销毁释放内存 |

### 2.2 性能优化策略

| 策略 | 触发时机 | 效果 |
|------|----------|------|
| 延迟加载控件 | 点击右侧区域 | 初始加载快 |
| 延迟创建菜单项 | 打开下拉菜单 | 点击加载快 |
| 即时保存 | 值变化时 | 退出无卡顿 |
| 延迟销毁 | 收起后30秒 | 内存可控 |

### 2.3 数据管理策略

**数据流向**：
```
配置文件 → 界面模块 → 折叠卡片 → 下拉框
    ↓          ↓          ↓         ↓
  默认值    当前值     显示值    选择值
    ↓          ↓          ↓         ↓
  ←←←←←←←← 即时保存 ←←←←←←←←←←←←←←←
```

**保存时机**：
| 时机 | 操作 | 说明 |
|------|------|------|
| 值变化时 | 即时保存 | 避免退出时批量保存 |
| 切换界面时 | 无需操作 | 已即时保存 |
| 退出应用时 | 无需操作 | 已即时保存 |

## 四、界面模块规范

### 4.1 模块结构规范

```python
# -*- coding: utf-8 -*-
"""
模块名称：XXX界面 | 设计思路：采用折叠卡片V2模式，右侧内容切换，延迟加载控件 | 模块隔离原则：不直接创建被调用模块的内容，不覆盖被调用模块的计算结果，用户指定变量除外
"""

import flet as ft

from 新界面.组件模块.折叠卡片 import CollapsibleCard
from 新界面.组件模块.通用功能容器 import GenericFunctionContainer
from 新界面.核心接口.主题提供者 import ThemeProvider
from 新界面.核心接口.界面配置 import InterfaceConfig
from .XXX配置 import XXX卡片配置


# *** 用户指定变量 - AI不得修改, 变量值必须生效 ***
USER_CARD_NAMES = ["卡片1", "卡片2", "卡片3"]
# *********************************


class XXXInterface:
    """XXX界面 - 折叠卡片V2模式"""
    
    @staticmethod
    def create(width: int=None) -> ft.Container:
        配置 = InterfaceConfig()
        
        cards = []
        for card_name in USER_CARD_NAMES:
            card_config = XXX卡片配置.get(card_name, {})
            if card_config:
                card = CollapsibleCard.create(
                    title=card_config.get("title", card_name),
                    icon=card_config.get("icon", "HOME"),
                    subtitle=card_config.get("subtitle", ""),
                    enabled=card_config.get("enabled", True),
                    controls_config=card_config.get("controls", []),
                    controls_per_row=card_config.get("controls_per_row", 6),
                    width=width,
                    on_save=lambda k, v: save_value(k, v),
                )
                cards.append(card)
        
        return GenericFunctionContainer.create(
            config=配置,
            title="XXX设置",
            icon="SETTINGS",
            cards=cards,
            width=width,
            expand=True,
        )


def save_value(config_key: str, value: str):
    """即时保存配置值"""
    # TODO: 实现配置保存逻辑
    print(f"保存: {config_key} = {value}")


# *** 调试逻辑 ***
if __name__ == "__main__":
    ThemeProvider.initialize(InterfaceConfig())
    ft.run(lambda page: page.add(XXXInterface.create()))
```

## 六、质量检查清单

### 6.1 模块独立性检查

| 检查项 | 要求 | 验证方法 |
|--------|------|----------|
| 独立运行 | 每个模块可独立运行 | `python XXX界面.py` |
| 独立配置 | 每个界面有独立配置文件 | 检查配置文件存在 |
| 独立测试 | 每个模块可独立测试 | 运行测试用例 |
| 接口清晰 | 通过 `__init__.py` 导出 | 检查导入是否正常 |

### 6.2 性能指标检查

| 指标 | 目标值 | 验证方法 |
|------|--------|----------|
| 初始加载时间 | ≤100ms | 计时测试 |
| 点击加载控件 | ≤100ms | 计时测试 |
| 切换界面响应 | ≤50ms | 计时测试 |
| 退出应用响应 | ≤50ms | 计时测试 |

### 6.3 代码规范检查

| 检查项 | 要求 |
|--------|------|
| 文档字符串 | 单行格式：`"""模块名称：XXX \| 设计思路 \| 模块隔离原则"""` |
| 用户变量区域 | 必须存在 |
| 导入顺序 | 标准库 → 第三方库 → 项目内部模块 |
| 函数参数 | 所有参数应有默认值，`=`两边不加空格 |

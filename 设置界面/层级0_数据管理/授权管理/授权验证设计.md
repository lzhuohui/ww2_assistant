# 授权验证系统设计文档

## 一、背景与约束

### 1.1 运行环境
- APK 发布
- 模拟器 + 手机双环境支持

### 1.2 约束条件
- 私人开发，无专门服务器
- 成本敏感（零成本）
- 技术实现简单
- 作者电脑可24小时开机（可选）

---

## 二、授权方案（最终确定）

### 2.1 定价模式：天次数制

#### 核心逻辑
- **天次数**：每天首次启动扣1次，当天重复启动不重复扣
- **基础授权**：1个账号 + 天数套餐
- **多账号授权**：购买天次数包，用于多账号模式

#### 基础授权套餐（单账号）

| 套餐 | 天数 | 价格 | 日均成本 |
|-----|------|------|---------|
| 日卡 | 1天 | 3元 | 3元/天 |
| 周卡 | 7天 | 18元 | 2.57元/天 |
| 月卡 | 30天 | 50元 | 1.67元/天 |
| 季卡 | 90天 | 120元 | 1.33元/天 |
| 年卡 | 365天 | 300元 | 0.82元/天 |

#### 多账号天次数包

| 套餐 | 天次数 | 价格 | 日均成本 |
|-----|-------|-----|---------|
| 体验包 | 15天次 | 25元 | 1.67元/天 |
| 标准包 | 30天次 | 45元 | 1.5元/天 |
| 超值包 | 60天次 | 80元 | 1.33元/天 |
| 大户包 | 150天次 | 180元 | 1.2元/天 |

#### 试用期
- 新用户首次运行自动获得 **7天免费试用**
- 试用期内功能完整，仅限1个账号

### 2.2 架构：离线授权 + 设备绑定 + 手动管理

```
┌─────────────────────────────────────────────────────────────┐
│                      用户购买流程                            │
├─────────────────────────────────────────────────────────────┤
│  1. 用户联系作者（微信/QQ）                                  │
│  2. 用户告知：套餐类型 + 设备码                              │
│  3. 用户付款（微信/支付宝）                                  │
│  4. 作者生成授权码 → 发送给用户                              │
│  5. 用户在APP中输入授权码激活                                │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      作者端工具                              │
├─────────────────────────────────────────────────────────────┤
│  授权码生成器.py                                             │
│  ├── 输入：套餐类型、设备码（可选）                          │
│  ├── 处理：生成加密授权码                                    │
│  └── 输出：授权码字符串                                      │
│                                                             │
│  订单记录.xlsx（手动维护）                                   │
│  ├── 用户信息、购买时间、套餐、授权码                        │
│  └── 用于后续查询和管理                                      │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 授权码设计

```
格式：XXXX-XXXX-XXXX-XXXX（16位）

编码内容（加密后）：
├── 授权类型：基础授权(0) / 天次数包(1)
├── 授权数量/天数：根据类型解释
│   ├── 基础授权：天数(1/7/30/90/365)
│   └── 天次数包：次数(15/30/60/150)
├── 设备码：绑定设备（可选，基础授权必填）
├── 生成时间：时间戳
├── 有效期：授权码本身有效期（防止长期未使用）
└── 校验码：防篡改
```

## 三、设备绑定策略

### 3.1 目标
- 一个授权码 = 一个设备（模拟器或手机）
- 模拟器可运行，但只能绑定一个实例
- 防止多开模拟器使用同一授权码

### 3.2 设备指纹组合

```python
设备指纹 = MD5(
    Android ID +        # 每个模拟器实例唯一
    设备型号 +          # 模拟器/手机型号
    设备品牌 +          # 模拟器品牌
    系统版本 +          # Android 版本
    首次安装时间 +      # APK安装时间
    随机种子            # 首次运行生成
)
```

### 3.3 模拟器检测特征

| 特征 | 模拟器值 | 真机值 |
|------|---------|--------|
| Build.FINGERPRINT | 包含generic/sdk | 厂商签名 |
| Build.MODEL | sdk/Emulator | 具体型号 |
| Build.MANUFACTURER | Genymotion/unknown | Samsung/Xiaomi等 |
| Build.HARDWARE | goldfish/ranchu | 实际硬件名 |

## 四、防护层级

### 4.1 第1层：设备指纹
- 多维度设备识别（Android ID + 硬件 + 环境）
- 克隆/复制后至少一个维度变化

### 4.2 第2层：授权存储
- 多重隐藏存储位置
- 数据交叉验证，篡改任一处即失效

存储位置：
```
1. 应用私有目录：/data/data/包名/files/.license
2. 外部存储隐藏：/sdcard/.android/.system/.cache
3. SharedPreferences：_sys_cache_data
4. 数据库隐藏表：app_data.db -> _system_config
```

### 4.3 第3层：时间防护
- 网络时间校验（NTP）
- 单调时间记录
- 时间回退检测

### 4.4 第4层：换绑限制
- 换绑次数限制：3次
- 冷却期限制：7天
- 宽限期机制：24小时

### 4.5 第5层：代码保护
- 代码混淆
- 关键逻辑加密
- 反调试检测

## 五、试用期规则

### 5.1 新用户试用

### 5.2 试用期存储
- 记录首次运行时间

## 六、授权流程

### 6.1 首次运行

```
检查本地授权文件
    │
    ├── 有基础授权 → 验证设备码 → 匹配 → 正常使用
    │                           └── 不匹配 → 提示"设备不匹配，请联系作者换绑"
    │
    └── 无授权 → 试用期模式（7天，单账号）
                    │
                    ▼
                 输入授权码
                    │
                    ├── 基础授权码 → 验证 → 绑定设备 → 保存
                    │                └── 无效 → 提示错误
                    │
                    └── 天次数包 → 验证 → 累加天次数 → 保存
                                     └── 无效 → 提示错误
```

### 6.2 日常启动

```
APP启动
    │
    ├── 检查基础授权
    │   ├── 有效 → 单账号模式可用
    │   └── 过期/无 → 单账号模式不可用
    │
    ├── 检查天次数
    │   ├── 有剩余 → 多账号模式可用
    │   │   └── 今日首次启动 → 扣减1次
    │   └── 无剩余 → 多账号模式不可用
    │
    └── 两者都无效 → 提示购买授权
```

### 6.3 换绑流程

```
用户申请新设备授权
    │
    ▼
用户提供：原授权码 + 新设备码
    │
    ▼
作者验证：换绑次数未超限（≤3次）
    │
    ▼
作者生成：新授权码（继承原有效期）
    │
    ▼
用户激活新授权码
    │
    ▼
原授权码失效
```

### 6.4 手动管理流程（作者端）

```
用户联系购买
    │
    ▼
确认套餐类型和价格
    │
    ▼
用户付款（微信/支付宝收款码）
    │
    ▼
用户提供设备码（APP中显示）
    │
    ▼
运行授权码生成器
    │
    ├── 基础授权：python 授权码生成器.py --type basic --days 30 --device <设备码>
    └── 天次数包：python 授权码生成器.py --type times --count 30
    │
    ▼
发送授权码给用户
    │
    ▼
记录到订单记录.xlsx
```

🔍 Gitee流水线功能开启检查指南

📋 检查前准备

请确保：

  • 已登录Gitee账号
  • 有仓库的管理权限
  • 使用Chrome/Firefox/Edge浏览器
  • 网络连接正常

1️⃣ 第一步：访问仓库设置

点击以下链接直接访问：
🔗 https://gitee.com/lzhuohui/ww2_assistant/settings

操作步骤：

  1. 点击上面的链接，打开仓库设置页面
  2. 确保已登录（右上角显示你的用户名）
  3. 在左侧菜单中找到 "功能设置"
  4. 点击进入功能设置

2️⃣ 第二步：开启流水线功能

在功能设置页面中：

📸 应该看到类似这样的界面：

功能设置
├── ☑ 代码质量
├── ☐ 代码扫描
├── ☐ 知识库
├── ☐ Wiki
├── ☐ 议题
├── ☐ 合并请求
├── ☐ 项目
├── ☐ Releases
├── ☐ 包管理
└── ☐ 流水线 ← 重点检查这里！

需要做：

  • 找到"流水线"选项
  • 勾选复选框：□ → ☑
  • 点击页面底部的"保存"按钮
  • 等待保存成功提示

3️⃣ 第三步：验证流水线功能

验证链接：
🔗 https://gitee.com/lzhuohui/ww2_assistant/actions

检查点：

✅ 成功标志：
  • 顶部菜单栏显示"流水线"标签
  • 能看到"Build APK for Gitee"工作流
  • 能看到构建历史列表
  • 能看到"运行流水线"按钮
❌ 失败标志：
  • 显示404页面
  • 提示"功能未开启"
  • 看不到"流水线"菜单
  • 页面空白或加载失败

4️⃣ 第四步：常见问题解决

问题A：完全看不到"流水线"选项

可能原因：

  • 仓库类型不支持（必须是公开仓库）
  • 账号权限不足
  • Gitee平台限制

解决方案：

  1. 确认仓库是公开的（不是私有）
  2. 联系仓库所有者获取管理员权限
  3. 联系Gitee客服：service@gitee.com
  4. 改用GitHub Actions

问题B：点击"流水线"显示404

可能原因：

  • 功能未正确开启
  • 浏览器缓存问题
  • 平台延迟

解决方案：

  1. 强制刷新页面：Ctrl+F5
  2. 清除浏览器缓存
  3. 换个浏览器尝试
  4. 等待5分钟后重试

问题C：已开启但仍失败

操作步骤：

  1. 查看构建 #7 的详细错误日志
  2. 点击"运行流水线"测试简单构建
  3. 提供错误截图给我分析

5️⃣ 第五步：备用方案

💡 如果Gitee流水线确实无法使用，可以：

方案1：使用GitHub Actions

执行以下命令切换回GitHub：

git push origin master

方案2：本地构建APK

运行本地构建脚本：

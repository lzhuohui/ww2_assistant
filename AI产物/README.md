# AI产物目录

## 目录说明

本目录专门用于存放AI处理过程中生成的所有文件和工具，以保持项目根目录的整洁性。

## 目录结构

```
AI产物/
├── 文档/           # 文档类文件
│   ├── 报告/       # 各类报告文档
│   ├── 方案/       # 设计方案文档
│   ├── 分析/       # 分析文档
│   └── 临时/       # 临时文档
├── 工具/           # 工具脚本
│   ├── 脚本/       # Python/Bash脚本
│   ├── 配置/       # 配置文件模板
│   └── 模板/       # 代码模板
├── 数据/           # 数据文件
│   ├── 原始/       # 原始数据
│   ├── 处理/       # 处理后数据
│   └── 备份/       # 数据备份
├── 日志/           # 日志文件
│   ├── 操作/       # 操作日志
│   ├── 错误/       # 错误日志
│   └── 审计/       # 审计日志
├── 临时/           # 临时文件
│   ├── 缓存/       # 缓存文件
│   └── 草稿/       # 草稿文件
└── 归档/           # 归档文件
    ├── 历史/       # 历史版本
    └── 废弃/       # 废弃文件
```

## 使用规范

### 文件命名规则

```
[类型]_[日期]_[描述]_[版本].[扩展名]

示例：
- 报告_20260308_环境配置_v1.0.md
- 脚本_20260308_字符统计_v1.0.py
- 数据_20260308_测试结果_v1.0.json
```

### 文件类型映射

| 文件类型 | 目标目录 | 命名前缀 |
|---------|---------|---------|
| 报告文档 | 文档/报告/ | 报告_ |
| 设计方案 | 文档/方案/ | 方案_ |
| 分析文档 | 文档/分析/ | 分析_ |
| Python脚本 | 工具/脚本/ | 脚本_ |
| 配置文件 | 工具/配置/ | 配置_ |
| 测试数据 | 数据/原始/ | 数据_ |
| 操作日志 | 日志/操作/ | 日志_ |
| 错误日志 | 日志/错误/ | 错误_ |
| 临时文件 | 临时/ | 临时_ |

## 管理工具

### 文件生成助手

`工具/脚本/文件生成助手.py`

用于帮助AI生成符合规范的文件。

### 目录清理脚本

`工具/脚本/目录清理脚本.py`

用于定期清理过期文件。

使用方法：
```bash
# 模拟运行（查看将要清理的文件）
python AI产物/工具/脚本/目录清理脚本.py --dry-run

# 实际清理
python AI产物/工具/脚本/目录清理脚本.py

# 查看存储统计
python AI产物/工具/脚本/目录清理脚本.py --stats
```

## 模板文件

- `工具/模板/报告模板.md` - Markdown报告模板
- `工具/模板/脚本模板.py` - Python脚本模板
- `工具/模板/配置模板.json` - JSON配置模板

## 清理策略

| 目录 | 保留天数 | 自动清理 |
|------|---------|---------|
| 临时/缓存/ | 7天 | 是 |
| 临时/草稿/ | 30天 | 是 |
| 日志/操作/ | 30天 | 是 |
| 日志/错误/ | 90天 | 是 |
| 数据/原始/ | 90天 | 否（手动确认） |

## 注意事项

1. 所有AI生成的文件必须存放在本目录下
2. 禁止在项目根目录创建AI产物文件
3. 遵循命名规范，便于管理和检索
4. 定期运行清理脚本，保持目录整洁
5. 重要文件请及时归档

## 环境要求

### 基础环境
- **操作系统**：Windows 11 64位专业版或企业版
- **Python版本**：3.9-3.11
- **虚拟环境**：专用Python虚拟环境
- **pip源**：国内镜像（清华大学）

### 模拟器环境
- **模拟器**：蓝叠模拟器9(64位)
- **分辨率**：2400×1080
- **DPI**：320

### 测试设备
- 至少2台不同品牌的真实Android设备
- 建议系统版本：Android 10及以上

## 配置步骤

### 虚拟环境创建

1. **创建虚拟环境**：
   ```bash
   python -m venv venv
   ```

2. **激活虚拟环境**：
   ```bash
   venv\Scripts\activate
   ```

3. **配置pip源**：
   ```bash
   pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
   ```

### 依赖安装

1. **安装Flet框架**：
   ```bash
   pip install flet
   ```

2. **生成依赖文件**：
   ```bash
   pip freeze > 前端/配置/requirements.txt
   ```

### 激活脚本

创建`前端/配置/activate_env.bat`脚本，用于快速激活虚拟环境。

使用方法：
```bash
前端/配置/activate_env.bat
```

### IDE配置

1. **配置Python解释器**：
   - 在Trae IDE中，选择`File > Settings > Project > Python Interpreter`
   - 点击`Add Interpreter > Add Local Interpreter`
   - 选择`Virtualenv Environment > Existing environment`
   - 浏览到`venv\Scripts\python.exe`
   - 点击`OK`完成配置

2. **安装Flet插件**：
   - 在Trae IDE中，选择`File > Settings > Plugins`
   - 搜索`Flet`插件并安装
   - 重启IDE以应用更改

### 界面测试

测试脚本位于：`AI产物/工具/脚本/脚本_20260308_Flet环境测试_v1.0.0.py`

该脚本实现包含文本、按钮、输入框的简单界面，用于验证Flet环境配置。

运行测试：
```bash
python AI产物/工具/脚本/脚本_20260308_Flet环境测试_v1.0.0.py
```

### 模拟器调试配置

1. **启动蓝叠模拟器**：
   - 打开蓝叠模拟器9
   - 进入设置，配置分辨率为2400×1080，DPI为320
   - 确保模拟器已启动并正常运行

2. **运行测试应用**：
   ```bash
   python AI产物/工具/脚本/脚本_20260308_Flet环境测试_v1.0.0.py
   ```

3. **验证应用运行**：
   - 检查应用是否正常启动
   - 测试输入框输入功能
   - 测试按钮点击功能
   - 测试界面更新功能

### 真机调试配置

1. **启用USB调试**：
   - 在Android设备上，进入`设置 > 关于手机`
   - 连续点击`版本号`7次，开启开发者模式
   - 进入`设置 > 开发者选项`，启用`USB调试`

2. **连接设备**：
   - 使用USB数据线连接设备到电脑
   - 在设备上授权USB调试

3. **验证设备连接**：
   ```bash
   adb devices
   ```

4. **运行测试应用**：
   ```bash
   python AI产物/工具/脚本/脚本_20260308_Flet环境测试_v1.0.0.py
   ```

## 故障排除

### 常见问题

1. **虚拟环境激活失败**：
   - 检查Python版本是否正确
   - 检查虚拟环境路径是否正确

2. **Flet安装失败**：
   - 确保pip源配置正确
   - 确保网络连接正常

3. **应用启动失败**：
   - 检查Flet版本是否兼容
   - 检查Python版本是否在3.9-3.11之间

4. **设备连接失败**：
   - 检查USB数据线是否连接正常
   - 检查USB调试是否启用
   - 检查ADB驱动是否安装正确

### 解决方案

1. **虚拟环境问题**：
   - 重新创建虚拟环境
   - 确保使用正确的Python版本

2. **Flet安装问题**：
   - 使用国内镜像源
   - 升级pip到最新版本

3. **应用启动问题**：
   - 查看错误信息，针对性解决
   - 检查依赖版本兼容性

4. **设备连接问题**：
   - 尝试更换USB数据线
   - 重启设备和电脑
   - 重新安装ADB驱动
